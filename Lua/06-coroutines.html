06 - 协程与异步编程

Lua 的协程（coroutine）是 stackful、非对称的协作式多任务机制。它不像 Go 的 goroutine 那样自动调度，不像 JS 的 async/await 那样基于 Promise。它是一种底层而灵活的工具——你可以用它搭建任何并发模型。

协程是什么

Lua 协程

一个协程是一个独立的 Lua 线程——它有自己完整的调用栈、局部变量和指令指针。它与主线程（和其他协程）共享全局状态。协程之间通过 yield（暂停）和 resume（恢复）协作切换。

特性Lua 协程操作系统线程Python asyncio
调度方式手动 yield/resume抢占式（OS 决定）事件循环 + await
并发性协作式（调用者决定何时切）真正并行（多核）协作式单线程
创建开销极小（Lua 级别）大（OS 级别）小（Python 级）
栈独立 Lua 栈独立 OS 栈独立 Python 栈
适合场景游戏逻辑、迭代器、状态机CPU 密集、阻塞 I/O网络 I/O、爬虫

协程 API 基础

-- 创建协程
local co = coroutine.create(function(param)
    print("协程启动，收到参数: " .. param)
    local received = coroutine.yield("第一次暂停")
    print("被唤醒，收到: " .. (received or "nil"))
    return "协程结束"
end)

-- resume 的第一个返回值是 bool（是否成功），后面是 yield 的返回值
local ok, yield_result = coroutine.resume(co, "hello")
-- 输出：协程启动，收到参数: hello
-- ok = true, yield_result = "第一次暂停"

-- 再次 resume，第二个参数传给 yield 的位置
local ok2, final_result = coroutine.resume(co, "world")
-- 输出：被唤醒，收到: world
-- ok2 = true, final_result = "协程结束"

-- 协程状态
print(coroutine.status(co))  --> dead

-- 协程生命周期：suspended → running → normal → dead
resume/yield 的数据流：resume 的额外参数 → 作为 yield 的返回值。yield 的参数 → 作为 resume 的返回值。这是双向通道，也是理解协程最难的地方。

生产者-消费者模式

-- 生产者：生成数据，每次 yield 一个
local function producer()
    for i = 1, 5 do
        coroutine.yield("item_" .. i)
    end
    return nil  -- 结束信号
end

-- 消费者：驱动生产者
local function consumer(prod)
    while true do
        local ok, item = coroutine.resume(prod)
        if not ok then
            error("Producer error: " .. item)
        end
        if item == nil then
            break
        end
        print("Consumed: " .. item)
    end
end

local co = coroutine.create(producer)
consumer(co)

协程实现游戏主循环调度

这是 Lua 在游戏中最经典的用法：每个 NPC、技能、任务都是一个协程，主循环每帧 resume 一小步。

-- 协程调度器
local Scheduler = {
    tasks = {},
}

function Scheduler:spawn(func)
    local co = coroutine.create(func)
    table.insert(self.tasks, {co = co, alive = true})
    return co
end

function Scheduler:update(dt)
    -- 每帧依次唤醒所有活着的协程一步
    for _, task in ipairs(self.tasks) do
        if task.alive then
            local ok, err = coroutine.resume(task.co, dt)
            if not ok then
                print("Task error: " .. err)
                task.alive = false
            elseif coroutine.status(task.co) == "dead" then
                task.alive = false
            end
        end
    end
    -- 清理死任务
    self:_cleanup()
end

-- 等待 N 秒（游戏内时间）
local function wait(seconds)
    local elapsed = 0
    while elapsed < seconds do
        local dt = coroutine.yield()
        elapsed = elapsed + dt
    end
end

-- 使用示例：一个 NPC 巡逻逻辑
Scheduler:spawn(function()
    print("NPC: 开始巡逻")
    while true do
        print("NPC: 走到 A 点")
        wait(2.0)                   -- 等 2 秒（分多个 frame 完成）
        print("NPC: 走到 B 点")
        wait(3.0)                   -- 等 3 秒
        print("NPC: 巡逻一圈完毕")
    end
end)

-- 模拟主循环：每帧 0.1 秒
for frame = 1, 60 do
    Scheduler:update(0.1)
end
协程 vs Unity Coroutine：Unity 的协程用 yield return new WaitForSeconds(2) 等待，调度由 Unity 引擎内部完成。Lua 协程需要你自己写调度器——这既是负担也是自由度。你可以定义自己的 wait 语义（等帧、等事件、等条件）。

用协程实现异步 I/O（无阻塞）

-- 非阻塞异步框架（简化版）
local Async = {
    pending = {},  -- 等待完成的协程
    sockets = {},   -- 待检查的 socket
}

-- 异步读取：发起读请求，挂起协程
function Async:async_read(socket, timeout)
    timeout = timeout or 30
    local co = coroutine.running()  -- 获取当前协程
    self.sockets[co] = {
        socket = socket,
        deadline = os.time() + timeout,
    }
    return coroutine.yield()  -- 挂起，等数据
end

-- 每帧检查：有数据的协程继续执行
function Async:poll()
    for co, info in pairs(self.sockets) do
        local data = info.socket:receive("*l")
        if data then
            self.sockets[co] = nil
            coroutine.resume(co, data)
        elseif os.time() > info.deadline then
            coroutine.resume(co, nil, "timeout")
            self.sockets[co] = nil
        end
    end
end

-- 协程包装器
function Async:run(func)
    local co = coroutine.create(func)
    coroutine.resume(co)
end

-- 使用示例：不需要 callback 地狱
Async:run(function()
    print("正在下载用户数据...")
    local user_data = Async:async_read(user_socket)
    if not user_data then
        print("下载失败")
        return
    end
    print("用户数据: " .. user_data)

    print("正在下载配置...")
    local config_data = Async:async_read(config_socket)
    print("配置: " .. config_data)
    -- 看起来是同步的代码，实际是异步执行的
end)
为什么不是 callback？ 上面的 async_read 让异步代码读起来像同步代码——没有嵌套回调，没有 Promise 链。协程保持了你写代码时的线性思维，而调度器在底层处理非阻塞。这是协程相较 callback 的核心优势。

coroutine.wrap：更简洁的语法

-- create + resume vs wrap
local co = coroutine.create(func)
coroutine.resume(co, arg)    -- 返回 ok, 返回值

-- wrap 返回一个函数，直接调用
local wrapped = coroutine.wrap(func)
wrapped(arg)                  -- 直接返回 yield 的值（不返回 ok）

-- wrap 的缺点：错误直接抛出，不会被 ok 捕获
local safe = coroutine.wrap(function()
    error("oops")
end)
-- safe()  -- 直接抛出错误，没有 pcall 保护！
wrap 的风险：coroutine.wrap 简洁但不能错误恢复——协程内部抛出的错误会穿透到调用者。生产代码建议用 coroutine.create + coroutine.resume，自己检查第一个返回值 ok。

本章小结

  • Lua 协程是 stackful、非对称、协作式——独立栈、手动 yield/resume
  • resume 的参数 → yield 的返回值；yield 的参数 → resume 的返回值
  • 游戏中最经典用法：每个逻辑单元是一个协程，主循环逐帧驱动
  • 协程可以让异步 I/O 代码读起来像同步代码，避免 callback hell
  • 生产代码用 create + resume（可错误恢复），不用 wrap（错误穿透）
