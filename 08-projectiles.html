弹幕系统

武器、敌怪、Boss 的表现力，很多时候都来自 Projectile，而不是 Item 本身。

这一章会改变你的理解方式 从这里开始，你需要把很多攻击行为拆成两层：物品负责“发起”，弹幕负责“飞行、碰撞、命中、消失”。只要这套思路建立起来，后面做武器和 Boss 会顺很多。

什么是 ModProjectile

在 Terraria 里，子弹、箭、火球、回旋镖、召唤物攻击体、部分 Boss 技能，底层大多都可以归到 Projectile。也就是说，弹幕不只是“会飞出去的东西”，而是一类通用战斗实体。

ModProjectile

ModProjectile 用来定义一个自定义弹幕的尺寸、持续时间、阵营归属、伤害类型、AI 行为、碰撞效果和绘制表现。它经常与 ModItem、ModNPC 联动使用。

因此，很多看起来像“武器特效”的东西，本质上其实是一个或多个弹幕在工作。Item 往往只负责 shoot 和 shootSpeed，真正的运动轨迹、追踪、爆炸、尾迹都在 Projectile 里。

一个最小可用弹幕

先从一个基础直线弹幕开始。你现在不用急着写花哨 AI，先把最基本的生存周期和阵营理解清楚。

using Terraria;
using Terraria.ID;
using Terraria.ModLoader;

namespace ExampleMod.Content.Projectiles;

public class StarBolt : ModProjectile
{
    public override void SetDefaults()
    {
        Projectile.width = 14;
        Projectile.height = 14;
        Projectile.friendly = true;
        Projectile.hostile = false;
        Projectile.DamageType = DamageClass.Magic;
        Projectile.penetrate = 1;
        Projectile.timeLeft = 180;
        Projectile.aiStyle = 1;
        AIType = ProjectileID.Bullet;
    }
}

这里只写了几行，但信息量很大：它是友方弹幕，不是敌对弹幕；它造成魔法伤害；最多贯穿一次；最多存活 180 帧；AI 复用了原版子弹风格。

非常实用的起步方式 刚开始写弹幕时，先学会“借原版行为”，再自己写 AI。也就是先用 aiStyle + AIType 或 CloneDefaults 快速做出一个稳定原型。

SetDefaults 里的关键字段

字段作用常见理解
width / height碰撞盒和绘制尺寸基础通常与贴图接近
friendly / hostile阵营归属打敌人还是打玩家
DamageType伤害职业决定吃哪些加成
penetrate可命中次数1 表示打中一次后结束
timeLeft剩余寿命单位是帧
aiStyle复用原版行为模板快速成型的捷径
tileCollide是否撞墙法球和穿墙技能常改这里

多数时候，Projectile 的第一版都应该先把这些字段调顺。它们决定了“这个东西大概怎么活着”，而不是“它有多帅”。

aiStyle 和 AIType：先借原版，再自定义

tModLoader 很鼓励你先复用原版弹幕行为。比如木箭、子弹、回旋镖、悠悠球，这些原版弹幕的基本逻辑都已经很成熟。你完全可以先套一个行为，再在其上做差异化。

public override void SetDefaults()
{
    Projectile.CloneDefaults(ProjectileID.EnchantedBoomerang);
    AIType = ProjectileID.EnchantedBoomerang;
}

这个写法很适合原型开发。你会先得到一枚“像魔法回旋镖一样飞”的弹幕，然后再逐步改伤害、特效、命中附带效果、尾迹和爆炸。

点击展开：aiStyle 和 AIType 到底怎么配合

aiStyle 决定使用哪一套原版通用 AI 逻辑；AIType 则更进一步，模仿某个具体原版弹幕的行为细节。很多时候二者配合使用，能让你快速获得“接近原版的稳定行为”。

AI 方法：真正的行为控制中心

当你不满足于原版模板，或者你想加入追踪、延迟爆炸、分裂、环绕、停顿再冲刺等独特行为时，就会开始重写 AI()。这个方法每帧都会执行一次，因此是弹幕“活着时在做什么”的核心入口。

public class PulsingOrb : ModProjectile
{
    public float Timer
    {
        get => Projectile.ai[0];
        set => Projectile.ai[0] = value;
    }

    public override void SetDefaults()
    {
        Projectile.width = 18;
        Projectile.height = 18;
        Projectile.friendly = true;
        Projectile.DamageType = DamageClass.Magic;
        Projectile.penetrate = 2;
        Projectile.timeLeft = 240;
        Projectile.tileCollide = false;
    }

    public override void AI()
    {
        Timer++;
        Projectile.rotation += 0.2f;

        if (Timer < 30)
        {
            Projectile.velocity *= 0.97f;
        }
        else
        {
            Projectile.velocity *= 1.02f;
        }
    }
}

这里用到了一个非常推荐的写法：把 Projectile.ai[0] 包成一个可读性更高的属性，比如 Timer。这样后面写逻辑时，你就不会老在脑子里翻译“ai[0] 现在是计时器还是目标索引”。

注意 AI() 每帧都跑，所以不要在里面无脑做重计算、重复查全地图、反复 new 大对象。弹幕一多，性能问题会非常明显。

Projectile.ai[] 和 localAI[] 应该怎么理解

Projectile.ai[] 是官方预留给你存储少量行为状态的数组。最常见的用法有计时器、阶段标记、目标索引、旋转方向、子行为参数等。localAI[] 则更适合放局部运行状态，不一定用于同步或外部传参。

ai[]

把它理解成“小型状态槽位”。与其写一堆魔法数字，不如给每个用途起属性名，例如 Timer、State、TargetIndex。

追踪弹幕示例

追踪弹幕是模组里非常常见的一类行为。它的关键不是“瞬间把方向改过去”，而是“逐渐修正速度向量”，这样视觉上才自然。

using Microsoft.Xna.Framework;
using Terraria;
using Terraria.ModLoader;

public class SeekingFlame : ModProjectile
{
    public override void SetDefaults()
    {
        Projectile.width = 16;
        Projectile.height = 16;
        Projectile.friendly = true;
        Projectile.DamageType = DamageClass.Magic;
        Projectile.penetrate = 1;
        Projectile.timeLeft = 240;
    }

    public override void AI()
    {
        NPC target = null;
        float maxDistance = 420f;

        foreach (NPC npc in Main.npc)
        {
            if (npc.CanBeChasedBy(this))
            {
                float distance = Vector2.Distance(Projectile.Center, npc.Center);
                if (distance < maxDistance)
                {
                    maxDistance = distance;
                    target = npc;
                }
            }
        }

        if (target != null)
        {
            Vector2 desiredVelocity = Projectile.DirectionTo(target.Center) * 10f;
            Projectile.velocity = Vector2.Lerp(Projectile.velocity, desiredVelocity, 0.08f);
        }

        Projectile.rotation = Projectile.velocity.ToRotation();
    }
}

这里用了 Vector2.Lerp 做平滑修正。它比“每帧直接把速度设为指向目标”要自然很多，因为后者通常会让弹幕拐弯太生硬，甚至像瞬移转向。

爆炸弹幕示例

另一类很常见的行为是“命中或到时爆炸”。这时候你往往会关心命中钩子、死亡前钩子，以及爆炸范围如何判定。

using Terraria;
using Terraria.Audio;
using Terraria.ID;
using Terraria.ModLoader;

public class BurstShell : ModProjectile
{
    public override void SetDefaults()
    {
        Projectile.width = 14;
        Projectile.height = 14;
        Projectile.friendly = true;
        Projectile.DamageType = DamageClass.Ranged;
        Projectile.penetrate = 1;
        Projectile.timeLeft = 120;
        Projectile.aiStyle = 1;
        AIType = ProjectileID.Bullet;
    }

    public override void OnHitNPC(NPC target, NPC.HitInfo hit, int damageDone)
    {
        Projectile.Kill();
    }

    public override void OnKill(int timeLeft)
    {
        SoundEngine.PlaySound(SoundID.Item14, Projectile.Center);

        for (int i = 0; i < 20; i++)
        {
            Dust.NewDust(Projectile.position, Projectile.width, Projectile.height, DustID.Torch);
        }
    }
}

这里先用 OnHitNPC 主动结束弹幕，再在 OnKill 里播放音效和生成尘埃。也就是说，爆炸视觉和命中判定并不一定非得同时写在一个方法里，拆开通常更清晰。

命中、碰撞与消失

Projectile 系统一般有三类关键节点：

  • OnHitNPC：命中 NPC 时做什么。
  • Colliding：自定义碰撞盒判定。
  • OnKill / PreKill：弹幕消失前后怎么处理。

如果你的弹幕形状比较特殊，比如激光、长鞭、扇形波、爆炸圈，就常常要重写 Colliding。因为默认矩形碰撞盒不足以表达这些形状。

点击展开：什么时候需要自己写 Colliding

当弹幕不再是一个普通小矩形时，就要开始考虑自定义碰撞。比如光束需要线段碰撞，环形爆炸可能需要半径碰撞，旋转刀刃可能需要跟着角度调整碰撞区域。

尾迹与视觉表现

一个普通弹幕和一个“看起来很有模组质感”的弹幕，差别往往就在尾迹、旋转、光照、命中反馈这些视觉层。tModLoader 提供了比较方便的尾迹缓存能力。

public override void SetStaticDefaults()
{
    ProjectileID.Sets.TrailCacheLength[Projectile.type] = 5;
    ProjectileID.Sets.TrailingMode[Projectile.type] = 0;
}

这段配置会让弹幕记录最近几帧的位置，后续你可以在绘制阶段用 Projectile.oldPos 画出拖尾。对于法球、飞刃、火焰、幽灵弹这类内容，尾迹非常加分。

视觉建议 不要一上来就把所有弹幕都做成五颜六色、拖尾拉满。真正舒服的视觉通常有重点：强弹幕更醒目，普通弹幕更克制。

如何把 Item 和 Projectile 接起来

别忘了，弹幕通常不是凭空出现的。最常见的入口还是武器 Item：

Item.noMelee = true;
Item.useStyle = ItemUseStyleID.Shoot;
Item.shoot = ModContent.ProjectileType<Content.Projectiles.SeekingFlame>();
Item.shootSpeed = 11f;
Item.DamageType = DamageClass.Magic;

这几行把一个魔法武器和自定义弹幕绑定起来。以后你看武器时要逐渐形成习惯：Item 只是发射端，Projectile 才是行为端。也因此，前面章节里做出的基础武器、配方和材料，到这里才真正开始“长出个性”——一旦换成你自己的 Projectile，武器就不再只是数值表，而是拥有独特行为的内容对象。

推荐的弹幕开发流程

1

先借原版行为

用 aiStyle 或 CloneDefaults 做出第一个能用版本。

2

确认基本生存参数

把宽高、阵营、寿命、是否撞墙、贯穿次数先调顺。

3

再写 AI 差异化

加入追踪、旋转、延迟、分裂、停顿等行为。

4

最后补视觉和命中反馈

尾迹、光照、爆炸、Dust、音效都放在行为稳定后再加。

本章小结

现在你已经掌握了 Projectile 的核心思路：先理解它是独立战斗实体，再用 SetDefaults() 定义生存规则，用 AI() 决定行为，用命中和消失钩子处理反馈。你也知道了如何借原版行为快速原型，以及如何用 ai[] 管理状态。
