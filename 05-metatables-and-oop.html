05 - 元表与面向对象

元表（metatable）是 Lua 的魔法引擎。没有 class 关键字？没关系——用 `__index` 实现继承。没有 operator overloading？`__add` 搞定。想让一个 table 表现得像一个函数？`__call`。元表让普通 table 拥有可编程的行为，这是 Lua 表达能力超越其体量的根源。

元表基础

元表

元表是一个普通的 table，附着在另一个 table（或 userdata）上，为其提供「元方法」——定义该对象在某些操作下的行为。每个 table/userdata 可以有自己独立的元表。

local t = {}
local mt = {}
setmetatable(t, mt)     -- 将 mt 设为 t 的元表
print(getmetatable(t))  --> table: 0x...

-- 只有 table 和 userdata 可以有元表
-- number/string/boolean/nil/function/thread 不能单独设置元表

__index：访问不存在键时的后备

__index 是最常用的元方法。当访问 table 中不存在的键时，Lua 会查元表的 __index。

-- __index 为函数：自定义查找逻辑
local defaults = {speed = 10, gravity = 9.8}
local t = {x = 100}

setmetatable(t, {
    __index = function(table, key)
        if key == "speed" then
            return 0   -- 默认速度为 0
        end
        return defaults[key]  -- 其他去 defaults 找
    end
})

print(t.x)      --> 100  (自己的字段)
print(t.speed)  --> 0    (__index 函数返回)
print(t.gravity) --> 9.8 (defaults 中的值)

-- __index 为 table：直接去该 table 查找（更常用）
local Animal = {
    eat = function(self) print(self.name .. " is eating") end,
    sleep = function(self) print(self.name .. " is sleeping") end,
}

local dog = {name = "Max"}
setmetatable(dog, {__index = Animal})

dog:eat()    --> Max is eating
dog:sleep()  --> Max is sleeping
冒号语法糖：obj:method(args) 等价于 obj.method(obj, args)。冒号自动将调用者作为第一个参数（通常叫 self）传入。OOP 模式下必须用冒号。

__newindex：拦截写入不存在的键

-- 只读 table
local data = {"a", "b", "c"}
setmetatable(data, {
    __newindex = function(t, k, v)
        error("Table is read-only", 2)
    end
})

-- data[1] = "x"  -- 报错！

-- __newindex 为 table：写入转发到该 table（类似代理）
local private_data = {score = 0}
local proxy = {}

setmetatable(proxy, {
    __index = private_data,     -- 读 → private_data
    __newindex = private_data,  -- 写 → private_data
})

proxy.score = 100
print(private_data.score)       --> 100

运算符元方法

Lua 支持为 table 定义运算符行为：

元方法对应运算符/操作
__add+
__sub-
__mul*
__div/
__mod%
__pow^
__unm一元负号 -
__idiv//
__band&
__bor|
__bxor~（位异或）
__bnot~（位非，一元）
__shl<<
__shr>>
__concat..
__len#
__eq==
__lt<
__le<=
 
-- 向量类 with 运算符
local Vector2 = {}
Vector2.__index = Vector2

function Vector2.new(x, y)
    return setmetatable({x = x or 0, y = y or 0}, Vector2)
end

function Vector2:__add(other)
    return Vector2.new(self.x + other.x, self.y + other.y)
end

function Vector2:__mul(scalar)
    return Vector2.new(self.x * scalar, self.y * scalar)
end

function Vector2:__tostring()
    return string.format("Vector2(%.1f, %.1f)", self.x, self.y)
end

function Vector2:__eq(other)
    return self.x == other.x and self.y == other.y
end

local a = Vector2.new(1, 2)
local b = Vector2.new(3, 4)
local c = a + b          --> Vector2(4.0, 6.0)
local d = a * 3          --> Vector2(3.0, 6.0)
print(a == b)            --> false
print(tostring(c))       --> Vector2(4.0, 6.0)
__eq 的特殊规则：两个对象需要拥有相同的 __eq 元方法才会触发自定义比较。如果两者元表不同，Lua 退化为引用比较（== 仅当两者是同一个对象时返回 true）。

__call：让 table 像函数一样被调用

-- 可调用对象
local Logger = {}
Logger.__index = Logger

function Logger.new(name)
    return setmetatable({name = name, count = 0}, Logger)
end

function Logger:__call(message)
    self.count = self.count + 1
    print(string.format("[%s] #%d: %s", self.name, self.count, message))
end

local log = Logger.new("Server")
log("Server started")          --> [Server] #1: Server started
log("Request received")        --> [Server] #2: Request received

-- __call 常用于实现状态机转化
local State = {}
function State.new(name, transition)
    return setmetatable({name = name, transition = transition}, {
        __call = function(self)
            print("Entering state: " .. self.name)
            if self.transition then
                return self.transition(self.name)
            end
        end
    })
end

local idle = State.new("Idle")
local running = State.new("Running")
idle()  --> Entering state: Idle

用元表实现 OOP：从原型继承到完整 class 框架

方案一：原型继承（最简单）

-- 定义一个「类」其实就是定义一个原型 table
local Animal = {
    name = "Unknown",
    speak = function(self)
        print(self.name .. " makes a sound")
    end,
}

-- 「实例化」就是创建一个以 Animal 为 __index 的 table
function Animal:new(o)
    o = o or {}
    setmetatable(o, self)
    self.__index = self
    return o
end

local dog = Animal:new({name = "Max"})
dog:speak()     --> Max makes a sound

-- 「继承」就是创建一个以父类为 __index 的子类
local Dog = Animal:new()
function Dog:speak()
    print(self.name .. " barks!")
end

local buddy = Dog:new({name = "Buddy"})
buddy:speak()   --> Buddy barks!

方案二：成熟的 class 框架（~20 行）

-- 一个简洁但完整的 class 系统
local function Class(classname, super)
    local cls = {}
    cls.__name = classname or "Unknown"
    cls.__super = super

    cls.__index = cls
    cls.__init = false  -- 标记未定义构造函数

    if super then
        setmetatable(cls, {
            __index = super,
            __call = function(c, ...)
                local obj = {}
                setmetatable(obj, c)
                if c.__init then c.__init(obj, ...) end
                return obj
            end
        })
    else
        setmetatable(cls, {
            __call = function(c, ...)
                local obj = {}
                setmetatable(obj, c)
                if c.__init then c.__init(obj, ...) end
                return obj
            end
        })
    end

    return cls
end

-- 使用示例
local Animal = Class("Animal")
function Animal:__init(name)
    self.name = name
end
function Animal:speak()
    print(self.name .. " makes a sound")
end

local Dog = Class("Dog", Animal)
function Dog:speak()
    print(self.name .. " barks!")
end

local buddy = Dog("Buddy")     -- 直接调用 class 创建实例
buddy:speak()                   --> Buddy barks!
print(buddy.name)               --> Buddy
选哪种？ 原型继承（方案一）代码最短，适合小型项目和原型。class 框架（方案二）提供构造函数、类型名、父子类关系查询，适合多人协作和大型项目。没有标准答案——Lua 就是让你自己选的。

其他常用元方法

-- __pairs / __ipairs：自定义迭代行为（5.2+）
local Range = {}
function Range.new(lo, hi)
    return setmetatable({lo = lo, hi = hi}, {
        __index = Range,
        __pairs = function(self)
            local i = self.lo - 1
            return function()
                i = i + 1
                if i <= self.hi then
                    return i, i * i   -- 值和它的平方
                end
            end
        end
    })
end

for i, sq in pairs(Range.new(1, 3)) do
    print(i, sq)  -- 1 1, 2 4, 3 9
end

-- __gc：终结器（Lua 5.2+ 支持 table/userdata 的 __gc）
-- 注意：必须在 setmetatable 时 __gc 就存在
local Resource = {}
function Resource.new(filename)
    local obj = {file = io.open(filename, "w")}
    setmetatable(obj, {
        __gc = function(self)
            print("Closing file: " .. filename)
            self.file:close()
        end
    })
    return obj
end

do
    local f = Resource.new("test.txt")
    f.file:write("hello")
    -- f 离开作用域
end
collectgarbage()  -- GC 时自动关闭文件

练习与思考

1

基础练习

用元表实现一个 DefaultDict——访问不存在的键时自动初始化为默认值（仿 Python 的 defaultdict）。

2

进阶练习

用 __index 和 __newindex 实现一个 LoggerProxy——对某个 table 的所有读写都自动记录日志。目标：不修改原代码就能追踪所有对 table 的操作。

3

设计思考

本章的 class 框架中，__call 用于 `cls(...)` 创建实例。如果不覆盖 __call，还有什么办法让类可以被调用创建实例？比较两种方案的优劣。

本章小结

  • 元表是普通的 table，通过元方法定义目标对象的行为
  • __index 是访问后备：函数（自定义逻辑）或 table（委托查找）
  • __newindex 拦截写操作，可用于只读代理或属性变更通知
  • 运算符元方法让 table 支持 +、*、==、.. 等操作
  • __call 让 table 可被调用，是实现 class 构造器的基础
  • Lua 的 OOP 不是内置的——是你用元表「拼」出来的，这正是 Lua 之美
