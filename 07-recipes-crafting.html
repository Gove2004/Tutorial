合成配方系统

让物品之间形成 progression：玩家为什么要收集你的材料，又如何把它们变成装备。

配方是内容生态的骨架 一个模组可以有很多漂亮物品，但如果它们之间没有合成路线，玩家只会觉得它们是孤立散落的素材。配方系统负责把材料、工作站、进度和奖励串起来。

配方写在哪里

最常见的写法，是在 ModItem 里重写 AddRecipes()。因为一个物品的配方通常可以跟着这个物品走：你想知道这把剑怎么合成，直接看这把剑的类就能找到。为了和前面章节形成贯穿练习，这里我们把前面“做出基础材料和世界物块”的内容继续往前推，真正合成出一把像样的模组武器。

using Terraria;
using Terraria.ID;
using Terraria.ModLoader;

namespace ExampleMod.Content.Items.Weapons;

public class StarBlade : ModItem
{
    public override void SetDefaults()
    {
        Item.width = 40;
        Item.height = 40;
        Item.damage = 32;
        Item.DamageType = DamageClass.Melee;
        Item.useTime = 20;
        Item.useAnimation = 20;
        Item.useStyle = ItemUseStyleID.Swing;
        Item.knockBack = 5.5f;
        Item.value = Item.buyPrice(gold: 1);
        Item.rare = ItemRarityID.Green;
        Item.UseSound = SoundID.Item1;
        Item.autoReuse = true;
    }

    public override void AddRecipes()
    {
        Recipe recipe = CreateRecipe();
        recipe.AddIngredient(ModContent.ItemType<Materials.StarOreChunk>(), 12);
        recipe.AddIngredient(ItemID.FallenStar, 3);
        recipe.AddTile(TileID.Anvils);
        recipe.Register();
    }
}

Recipe

Recipe 对象描述一条合成规则：输出什么、需要哪些材料、在哪个合成站旁边、是否有额外条件。最后必须调用 Register() 才会真正注册进游戏。

CreateRecipe、AddIngredient、AddTile、Register

一条最普通的配方通常由四步组成：创建配方、添加材料、指定合成站、注册。理解这四步以后，大部分配方都只是组合变化。

1

CreateRecipe

以当前物品为输出创建配方。可以传入数量，比如 CreateRecipe(5) 表示一次合成 5 个。

2

AddIngredient

添加材料，可以是原版物品 ID，也可以是 ModContent.ItemType<T>()。

3

AddTile

指定合成站，例如工作台、熔炉、铁砧，或者你自己的自定义家具。

4

Register

把这条配方提交给配方系统。忘了这一步，配方就不会出现。

最常见错误 写完材料和合成站后忘记 recipe.Register()。代码不一定报错，但游戏里就是找不到配方。

输出多个物品

消耗品、弹药、砖块这类物品通常一次合成多个。你可以在 CreateRecipe 里传入输出数量。

public override void AddRecipes()
{
    Recipe recipe = CreateRecipe(25);
    recipe.AddIngredient(ItemID.StoneBlock, 1);
    recipe.AddIngredient(ModContent.ItemType<Materials.StarOreChunk>(), 1);
    recipe.AddTile(TileID.WorkBenches);
    recipe.Register();
}

别小看输出数量。它直接影响玩家刷材料的压力。砖块、弹药、平台如果一次只出一个，玩家会觉得合成体验很折磨。

RecipeGroup：替代材料组

Terraria 里经常有“任意铁锭”“任意木材”这种设计。你不希望玩家必须用某一种特定材料，也不希望为每种材料写一条配方。这个时候就需要 RecipeGroup。

public override void AddRecipes()
{
    Recipe recipe = CreateRecipe();
    recipe.AddRecipeGroup(RecipeGroupID.IronBar, 8);
    recipe.AddIngredient(ItemID.FallenStar, 2);
    recipe.AddTile(TileID.Anvils);
    recipe.Register();
}

RecipeGroupID.IronBar 代表“铁锭或铅锭”。使用这种写法可以让配方更符合原版体验，也减少玩家因为世界矿物差异而卡进度。

点击展开：什么时候该用 RecipeGroup
  • 同一类原版材料有多个世界替代版本，如铁/铅、铜/锡。
  • 你想让玩家使用任意木材、任意宝石、任意邪恶材料。
  • 你的模组自己定义了多个等价材料，希望它们能互相替代。

自定义合成站

上一章我们做过自定义工作台。只要它是一个 ModTile，就可以被配方引用为合成站。

public override void AddRecipes()
{
    Recipe recipe = CreateRecipe();
    recipe.AddIngredient(ModContent.ItemType<Materials.StarOreChunk>(), 20);
    recipe.AddIngredient(ItemID.HellstoneBar, 4);
    recipe.AddTile(ModContent.TileType<Tiles.Furniture.StarWorkbenchTile>());
    recipe.Register();
}

自定义合成站能让模组 progression 更有辨识度。比如你可以要求玩家先制作“星辉熔炉”，再用它加工特殊矿石。这样玩家会感觉自己进入了模组自己的科技树。

设计建议 自定义合成站不要滥用。每加一个新合成站，就等于给玩家多一个必须记住的地点。它最好代表一个明确阶段，而不是为了“显得高级”才加。

条件合成

有些配方不应该永远可用。比如只在血月期间出现、只在困难模式后出现、只在击败某个 Boss 后出现。条件合成可以让配方成为世界进度的一部分。

public override void AddRecipes()
{
    Recipe recipe = CreateRecipe();
    recipe.AddIngredient(ItemID.DemoniteBar, 10);
    recipe.AddIngredient(ItemID.Lens, 3);
    recipe.AddTile(TileID.Anvils);
    recipe.AddCondition(Condition.DownedEyeOfCthulhu);
    recipe.Register();
}

条件合成的核心价值不是“隐藏配方”，而是控制玩家什么时候进入下一阶段。你可以把它想成一种轻量 progression gate。

多条配方：让玩家有多种路线

一个物品可以有多条配方。比如你想让玩家可以用腐化路线或猩红路线都合成同一件装备，就可以注册两条配方。

public override void AddRecipes()
{
    Recipe corruptionRecipe = CreateRecipe();
    corruptionRecipe.AddIngredient(ItemID.DemoniteBar, 10);
    corruptionRecipe.AddIngredient(ItemID.ShadowScale, 5);
    corruptionRecipe.AddTile(TileID.Anvils);
    corruptionRecipe.Register();

    Recipe crimsonRecipe = CreateRecipe();
    crimsonRecipe.AddIngredient(ItemID.CrimtaneBar, 10);
    crimsonRecipe.AddIngredient(ItemID.TissueSample, 5);
    crimsonRecipe.AddTile(TileID.Anvils);
    crimsonRecipe.Register();
}

这种写法比硬要求某一世界材料更友好。玩家不会因为开到不同世界邪恶类型而被迫去别的世界搬材料。

配方设计的平衡思路

推荐做法不推荐做法
材料来源覆盖探索、战斗、采矿多个行为所有材料都来自同一种怪，导致刷怪枯燥
关键装备需要明确阶段门槛强力装备开局就能轻易合成
替代世界材料使用 RecipeGroup强迫玩家必须拥有另一个世界的矿物
消耗品一次合成多个弹药、砖块一次只合成一个

实战：为一个小型模组建立第一条路线

假设你已经有星辉矿石、星辉锭、星辉剑、星辉工作台。一个合理的早期路线可以这样设计：

1

采矿阶段

玩家在地下挖到 StarOreItem。

2

冶炼阶段

若干星辉矿石在熔炉旁合成星辉锭。

3

工具阶段

星辉锭合成星辉工作台，解锁模组自己的合成站。

4

奖励阶段

玩家用星辉工作台合成第一把模组武器。

这条路线不复杂，但它已经让你的内容有了“从材料到装备”的闭环。后续你可以继续叠加 Boss 掉落、事件材料、生物群落资源，让路线变得更丰富。

本章小结

配方系统的 API 很简单，但设计价值很大。它不仅负责“怎么合成”，还负责“玩家怎么推进”。你现在应该能写基础配方、多输出配方、替代材料配方、自定义合成站配方，以及带条件的 progression 配方。
