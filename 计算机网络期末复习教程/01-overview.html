第1章 概述

这一章是整个计算机网络的绪论课。就像学习一门新语言先了解历史背景和基本语法一样，学习计算机网络也需要先建立整体认知：它为什么出现、由什么组成、怎么工作、性能如何衡量。本章是后续所有章节的理论基石，考试占比约15%。

1.1 计算机网络基本概念：为什么需要计算机网络？

从单机到网络：一个简单故事

想象20世纪80年代的办公室。张三在电脑上写好年度报告，想交给隔壁办公室的李四。怎么办？最常见的办法：把文件拷贝到一张软盘上（1.44MB方形塑料盘），然后走过去把软盘递给李四。俗称"跑腿网络"（sneakernet）——用鞋底传输数据。

如果文件很大（几百MB的演示文稿），一张软盘装不下得拆成好几张。如果两人在不同城市就得邮寄。如果需要实时更新（如股票报价），软盘拷贝完全不可行。

显然我们需要更好的方式——让计算机之间直接通过线路连接，数据以电/光信号传输，瞬间到达。这就是计算机网络的原始动机。

思考：计算机网络要解决的核心问题是什么？
答案：让地理位置不同的计算机能相互通信、共享资源。

计算机网络的定义

计算机网络是将地理位置不同的具有独立功能的多台计算机及其外部设备，通过通信线路连接起来，在网络操作系统、网络管理软件及网络通信协议的管理和协调下，实现资源共享和信息传递的计算机系统。

三要素：设备 + 链路 + 协议

从定义中提炼出计算机网络的三要素，缺一不可：

要素类比说明举例
计算机设备说话的人至少两台独立功能的计算机PC、服务器、手机
通信链路电话线连接计算机的物理通道双绞线、光纤、无线电波
网络协议语言规则通信双方必须遵守的规则TCP/IP、HTTP、FTP

比方：两个人打电话需要电话机（设备）、电话线（链路）和同一种语言（协议），缺一不可。

两个核心维度：带宽和延迟

评价网络主要看两个指标：

  • 带宽（Bandwidth）：水管粗细。越粗单位时间流过的水越多。
  • 延迟（Latency）：水从源头到目的地的时间。即使水管很粗，距离远也需要时间。
比喻网络术语影响什么
水管粗细带宽单位时间能传多少数据
水流速度延迟/时延数据一端到另一端要多久
经验：下载大文件带宽决定速度；网页打开和游戏更关心延迟（ping值）。

理解检查：在线射击游戏画面卡顿、人物瞬移，是带宽不够还是延迟太高？
（答案：延迟太高。游戏数据量不大但需极低延迟。）

1.2 互联网发展史

为什么讲历史？

理解互联网发展史，能帮你回答常见问题："为什么互联网采用现在的结构？"答案在历史中——互联网不是一次性设计出来的，而是不断演进的结果。

第一阶段：ARPANET的故事（1969年）

20世纪60年代，美国国防部高级研究计划局（ARPA）想建立一个去中心化的军事通信网络。传统电话网是中心化的——中心被摧毁全网瘫痪。冷战背景下这显然不靠谱。

1969年，ARPANET诞生，最初4个节点：

节点机构地点
1UCLA（加州大学洛杉矶分校）洛杉矶
2SRI（斯坦福研究院）门洛帕克
3UCSB（加州大学圣塔芭芭拉分校）圣塔芭芭拉
4犹他大学盐湖城

1969年10月29日，UCLA学生试图远程登录SRI，输入"LOG"后系统崩溃——这是互联网的第一次通信，只成功三个字母，但开启了新纪元。

1983年1月1日，ARPANET正式采用TCP/IP协议族，这天被视为互联网的正式诞生日。TCP/IP像统一了度量衡，让不同厂商的设备能互相通信。

第二阶段：三级结构互联网（1985-1993）

1985年，美国国家科学基金会（NSF）建立NSFNET，形成三级结构：

1

主干网

国家级高速网络，连接各主要节点。速度从56kbps升级到T3（44.736Mbps）。

2

地区网

覆盖一个地理区域（如一个州），连接区域内各类网络。

3

校园网/企业网

最底层用户网络，连接大学、研究机构或企业。

第三阶段：多层次ISP结构（1994年至今）

1994年互联网全面商业化。NSFNET退役，由ISP运营。形成多层次ISP结构：

层次角色示例
骨干ISP（Tier 1）国际级骨干网，互相对等连接AT&T、中国电信骨干网
区域ISP（Tier 2）向骨干ISP购买带宽，服务下游各地运营商分公司
本地ISP（Tier 3）直接面向家庭和企业长城宽带、小区宽带
考试提示：互联网三阶段（ARPANET -> 三级结构 -> 多层次ISP）常考选择题和简答题，记住关键年份和特征。

互联网标准化组织

组织全称类比职责
ISOCInternet Society（互联网协会）董事会领导机构，负责总体方向
IETFInternet Engineering Task Force（互联网工程任务组）工程师团队制定技术标准，发布RFC
IRTFInternet Research Task Force（互联网研究任务组）研究团队长期前沿技术研究

RFC文档类型

RFC（Request For Comments，请求评论）——名字体现互联网开放协作精神。并非所有RFC都是标准：

类型说明
标准类经IETF审查的正式标准，编号STD xxxx
草案类讨论中的提案，可能发展为标准
实验类用于实验目的
信息类提供背景信息、指南
历史类已被替代或废弃

标准确立流程：互联网草案（有效6月） -> 建议标准 -> 草案标准（需2个独立实现验证） -> 互联网标准。

理解检查：RFC中的"Request For Comments"体现了什么理念？
（答案：开放、协作、多方参与——标准制定不是某公司说了算，而是公开讨论、同行评审。）

1.3 互联网的组成

从工作方式角度，互联网分为边缘部分和核心部分：

边缘部分核心部分
组成主机（端系统）路由器 + 通信链路
类比城市住户高速公路系统
功能运行应用程序，为用户服务连通性和数据转发
示例PC、服务器、手机路由器、光纤骨干网

边缘部分：两种通信模式

C/S模式

场景：打开浏览器访问百度。你电脑是客户，百度是服务器。服务器7x24小时开机等客户来"敲门"。

  • 角色：客户请求服务，服务器提供服务
  • 特点：客户间不直接通信，服务器固定IP
  • 优点：管理集中、安全性好
  • 缺点：服务器压力大——双十一淘宝每秒处理几十万请求
  • 典型：Web、电子邮件、FTP

P2P模式

场景：用BT下载电影，从其他用户下载片段，同时上传给别人。每台电脑既是客户又是服务器。

  • 角色：节点地位对等，无固定服务器
  • 特点：用户越多速度越快（上传者变多了）
  • 优点：可扩展性好，无单点故障
  • 缺点：管理困难、安全性低
  • 典型：BT下载、微信文件传输、区块链
比较：实际应用常混合两种模式。微信聊天走C/S（消息先到腾讯服务器），传大文件走P2P（直接手机间传输）。

理解检查：BT下载为何"下载的人越多速度越快"而C/S模式中"访问的人越多服务器越慢"？
（答案：P2P中每个下载者也上传，总带宽越大；C/S所有请求压在一台服务器。）

核心部分：路由器的作用

核心部分的主角是路由器——网络中的"交通警察"，查看数据包目的地址，决定转发到哪条链路。核心部分就是大量路由器互连的网状网络。

1.4 三种交换方式（必考对比）

高频考点：三种交换方式对比几乎每年必考，可能是选择、简答或论述题。务必理解原理和优缺点。

交换方式解决同一个问题：数据怎么从源主机传送到目的主机？

电路交换（Circuit Switching）——打电话的方式

WHY：电话通信设计。如果每次说话前先"抢占线路"，说完释放，通话无法进行。所以先建立专用通路，独占直到通话结束。

WHAT：通信前建立一条端到端的专用物理通路，通路被双方独占。

HOW：三个阶段：

1

建立连接（拨号）

源端发出连接请求，中间节点逐段分配资源，建立端到端专用通路。

2

通信（通话）

数据沿路径传输，按序到达，无丢失。线路被独占——不说话别人也不能用。

3

释放连接（挂断）

一方发出请求，节点逐段释放资源。

优缺点：优点——时延小、有序、无冲突。缺点——建连耗时、线路独占利用率低、不适于突发数据。

报文交换（Message Switching）——寄信的方式

WHY：电路交换独占太浪费——计算机通信是突发性（bursty）的，多数时间无数据，偶尔爆发。让多台计算机共享线路？报文交换是方案之一。

WHAT：采用存储转发。发送方把整个数据（"报文"）交给中间节点，节点存储、检查目的地址、转发。

缺点：报文可能很大（几百MB视频文件），节点需要大缓存，转发时延巨大。

分组交换（Packet Switching）——互联网采用的方式

WHY：报文交换缓存压力大。把大报文切成小片，每片独立转发。

WHAT：将大报文分割成较小分组（packet），每个分组包含源/目的地址等控制信息（首部），通过路由器逐跳转发。

分组交换核心：存储转发

路由器收到完整分组后，存储到缓冲区，查目的地址和路由表，转发到下一跳。不同分组可能走不同路径。这就是"存储转发"的精髓。

优缺点：优点——共享线路利用率高、缓存压力小。缺点——分组可能乱序、需首部开销、排队时延不确定。

完整计算对比：A发100MB文件给B

假设A发100MB文件给B，途径3段链路，每段1Mbps。分组交换把文件分成1000个分组，每个0.1MB（忽略首部开销）。

点击查看详细计算

电路交换：

总时间 = 连接建立时间 + 发送时间

发送时间 = 100MB × 8 / 1Mbps = 800s

假设连接建立时间 = 0.5s

总时间 = 800.5s


分组交换：

总时间 = (分组大小/带宽)×链路数 + (分组数-1)×(分组大小/带宽)

= (0.1×8/1)×3 + 999×(0.1×8/1)

= 2.4 + 799.2 = 801.6s


报文交换：

总时间 = (报文大小/带宽)×链路数

= 800 × 3 = 2400s

交换方式总时间独占线路适用场景
电路交换800.5s是（利用率低）电话网
报文交换2400s否早期电报（已淘汰）
分组交换801.6s否（高效共享）Internet
关键洞察：分组交换（801.6s）与电路交换（800.5s）总时间接近，但电路交换空闲时也独占线路，分组交换允许多用户共享——整体效率高得多。这就是计算机网络选择分组交换的根本原因。

理解检查：什么情况下分组交换总时间明显大于电路交换？
（答案：分组非常多时，每个分组的发送和存储转发时延累积起来很大。）

1.5 计算机网络的分类

按覆盖范围分类

类型范围类比场景
PAN~10米个人活动范围蓝牙耳机连手机
LAN~1公里一栋楼办公室网络、WiFi
MAN10~100公里一座城市城市政务网
WAN几百~几千公里全国/全球Internet
口诀：PAN个人、LAN局部、MAN城市、WAN广域。LAN属一个组织，WAN由ISP运营。

按拓扑结构分类

星型拓扑

节点通过独立线路连到中心节点（如交换机）。类比：车轮。

优点：单节点故障不影响其他。缺点：中心是单点故障。应用：现代以太网。

总线型拓扑

节点连在一条共享总线。类比：树上的鸟。

优点：简单、成本低。缺点：总线故障全瘫痪，节点越多冲突越多。应用：早期以太网。

环型拓扑

节点构成闭合环，数据沿环传输。类比：接力赛。

优点：时延确定。缺点：单节点故障导致全网中断。应用：FDDI。

树型拓扑

多级星型分层。类比：公司组织架构。

优点：分级管理、利于扩展。缺点：根节点依赖强。应用：校园网。

网状型拓扑

节点间有多条路径。类比：蜘蛛网。

优点：极高可靠性，无单点故障。缺点：成本高、布线复杂。应用：互联网骨干网。

理解检查：家里WiFi是哪种拓扑？
（答案：星型——路由器是中心节点。）

1.6 性能指标（必考计算！）

必考计算题：性能指标计算每年期末必考，务必掌握每个公式和单位换算。

速率（Rate）

WHY：需要知道网络传输速度，就像买车关注最高时速。

WHAT：速率（比特率）= 单位时间传输的比特数。

关键提醒：速率单位用10的幂次（十进制），数据量单位用2的幂次（二进制）。常搞错！

单位换算（十进制）
1 kbps10³ bps = 1000 bps
1 Mbps10⁶ bps = 1,000,000 bps
1 Gbps10⁹ bps
1 Tbps10¹² bps
注意：1 KB = 1024 B = 2¹⁰ B，但 1 kbps = 1000 bps = 10³ bps。计算时先统一单位！

带宽（Bandwidth）

网络通信线路的最大传输能力，即单位时间能通过的最高数据率。带宽是上限。

例：宽带套餐"100Mbps"，理论上每秒最多传100兆比特（约12.5MB）。

吞吐量（Throughput）

带宽是理论最大值（限速），吞吐量是实际测量值（实际速度）。吞吐量通常小于带宽。

时延（Delay/Latency）——核心考点

数据从一端传到另一端的总时间由四部分组成：

总时延 = 发送时延 + 传播时延 + 处理时延 + 排队时延

时延类型类比公式
发送时延灌水进水管的时间数据长度 / 发送速率
传播时延水从一端流到另一端的时间信道长度 / 传播速度
处理时延检查包裹决定转寄地址取决于路由器性能
排队时延排队等待处理取决于网络拥塞
易错：发送时延是"放数据到线路"的时间，关心数据大小。传播时延是"数据在线路上跑"的时间，关心距离远近。
记忆：发送时延看数据，传播时延看距离。

例题

例题1：综合时延计算

题目：数据10MB，带宽100Mbps，链路1000km，光速2.0×10⁸ m/s。忽略处理时延和排队时延，求总时延。

解：统一单位：10MB = 10×1024×1024×8 = 83,886,080 bit

发送时延 = 83,886,080 / 100,000,000 ≈ 0.839 s

传播时延 = 1000×10³ / (2.0×10⁸) = 0.005 s = 5 ms

总时延 ≈ 0.844 s（发送时延是瓶颈）

例题2：传播时延占主导

题目：跨洋光缆8000km，100Gbps，传64字节控制分组。光速2.0×10⁸m/s。

解：发送时延 = 64×8/10¹¹ = 5.12 ns（可忽略）

传播时延 = 8000×10³/(2.0×10⁸) = 0.04 s = 40 ms

结论：带宽极大、数据极小时，传播时延（距离）是主要因素。纽约到上海40ms的传播时延无法避免。

时延带宽积

时延带宽积 = 传播时延 × 带宽

表示链路中已发送但未到达目的地的最大比特数。类比：水管中同时流的水量。

往返时间 RTT（Round-Trip Time）

RTT = 2 × 单向传播时延 + 处理时间

日常体验：游戏里的"ping值"就是RTT。ping值50ms意味着发数据到服务器再收到回复需要50ms。

利用率

信道利用率 = 有数据通过的时间 / 总时间

时延与利用率关系：D = D₀ / (1 - U)

重要：U接近1时，D趋近无穷大。网络越拥堵时延指数级恶化。利用率控制在50%~70%较合理。

1.7 计算机网络体系结构（超级重点）

核心考点：OSI七层模型和TCP/IP四层模型是必考内容，务必掌握各层名称、功能和对比。

为什么分层？——邮政系统类比

邮政系统网络层次

写地址邮政编码网络层

二、计算题

  1. 10Mbps以太网传500字节数据帧，信号速度2.0×10⁸m/s，信道长1000m。求：(1)发送时延 (2)传播时延 (3)总时延 (4)时延带宽积
  2. 1Mbps链路长5km，信号速度2.0×10⁸m/s，传1000字节数据帧，求总时延。
  3. 数据长10MB，带宽100Mbps，链路长2000km，信号速度2.0×10⁸m/s，求总时延。
答案

第1题：发送时延 = (500×8)/10×10⁶ = 0.0004s = 0.4ms；传播时延 = 1000/2.0×10⁸ = 5μs；总时延 = 0.405ms；时延带宽积 = 5×10⁶×10×10⁶ = 50bit

第2题：发送时延 = (1000×8)/10⁶ = 0.008s = 8ms；传播时延 = 5000/2.0×10⁸ = 0.025ms；总时延 = 8.025ms

第3题：发送时延 = 10×1024×1024×8/10⁸ = 0.839s；传播时延 = 2000×10³/2.0×10⁸ = 0.01s = 10ms；总时延 ≈ 0.849s
