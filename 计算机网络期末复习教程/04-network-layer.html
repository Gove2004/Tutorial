第4章 网络层

网络层是TCP/IP体系结构中最核心、最重要的一层。它负责将分组从源主机送达目的主机，提供端到端的通信服务。本章是期末考试中分值最高、内容最多的一章——IP地址、子网划分、CIDR和路由协议是必考内容。

4.1 网络层功能

为什么需要网络层？

数据链路层只负责相邻节点之间的通信——比如从你的电脑到家里的路由器。但互联网上的两台计算机可能相隔千里，中间经过几十台路由器。谁来负责把数据从源主机一站一站送到目的主机？这就是网络层的任务。

打个比方：数据链路层像是你从家走到地铁站（相邻节点），网络层则是规划从你家到北京天安门的完整路线——先坐哪路公交、换乘哪条地铁线、出站后走哪个出口。网络层负责端到端的全局路径规划。

网络层的核心功能

  • 路由选择（Routing）：确定分组从源到目的所经过的最佳路径。由路由算法决定，基于网络拓扑和状态信息。
  • 分组转发（Forwarding）：路由器根据路由表将分组从输入端口转发到适当的输出端口。转发是路由选择的执行阶段。
  • 拥塞控制：当网络负载超过容量时，采取相应措施缓解或避免拥塞。
  • 网络互联：将不同类型的网络连接起来，实现跨网络通信。

虚电路服务 vs 数据报服务

网络层可以向传输层提供两种服务方式。Internet的网络层（IP协议）采用的是数据报服务——每个分组独立寻路，不预先建立连接。

为什么Internet选择数据报服务？ 设计初衷是为了在战争或灾难下维持通信——数据报服务无连接的特性使得单个节点故障不影响整体通信，分组可以绕开故障节点独立寻路。此外，将复杂性放在端系统而非网络核心中，符合"端到端原则"。

一句话：Internet的设计哲学是"聪明的终端，傻的网络"——网络层只做尽力送达，可靠传输交给端系统去实现。

4.2 分类IP地址（必记！）

为什么需要IP地址？

MAC地址是设备出厂时固化的标识，没有层级结构——就像人的身份证号，无法从中看出你住哪个城市。而IP地址是有层次结构的：它由网络号（你在哪个网络）和主机号（你在该网络中的哪台设备）组成。路由器只需根据网络号转发分组，不需要知道每台主机的具体位置，这大大减小了路由表的规模。

IP地址结构

IP地址是32位的二进制数（IPv4），通常用点分十进制表示，如 192.168.1.1。

IP地址 ≈ { <网络号> , <主机号> }

这种两级结构是路由的基础——先找到网络，再找到主机。

4.3 ARP协议

为什么需要ARP？

主机A知道主机B的IP地址（比如192.168.1.2），但在同一局域网内通信时，数据链路层的帧需要使用MAC地址来标识目的设备。光知道IP地址还不够，需要把IP地址解析成对应的MAC地址——这就是ARP（地址解析协议）的工作。

简单来说：ARP解决的是"我知道你的IP，但我需要知道你的MAC地址"的问题。

ARP工作流程：4步

1

查ARP缓存

主机在本地ARP缓存表中查找目的IP对应的MAC地址。ARP缓存存储了最近通信过的IP-MAC对应关系，每一条目有生存时间（通常2~20分钟），超时后删除。

2

广播ARP请求

如果缓存未命中，主机在本局域网内广播一个ARP请求帧。目的MAC地址为全1（FF-FF-FF-FF-FF-FF），内容："谁的IP是X.X.X.X？请告诉我你的MAC地址。"

3

单播ARP响应

局域网内所有主机都收到该请求，但只有IP匹配的主机B处理它。B发送一个ARP响应帧（单播），内容："IP地址X.X.X.X是我的，我的MAC地址是XX-XX-XX-XX-XX-XX。"

4

更新缓存并通信

A收到ARP响应后，将IP-MAC映射写入ARP缓存，然后用正确的MAC地址封装数据帧进行通信。

ARP交互过程可视化：

主机A (IP:192.168.1.1)          主机B (IP:192.168.1.2)
      |                              |
      |  [1] 查缓存，未找到           |
      |                              |
      |  [2] 广播ARP请求              |
      | ---→ "谁的IP是192.168.1.2？" →|
      |     (目的MAC=FF-FF-FF-FF-FF-FF) |
      |                              |
      |  [3] 单播ARP响应              |
      | ←-- "192.168.1.2 = AA:BB:CC:DD:EE:FF" ---|
      |                              |
      |  [4] 更新缓存，正常通信        |
      | ---→ 目的MAC=AA:BB:CC:DD:EE:FF →|
ARP欺骗（ARP Spoofing）：攻击者伪造ARP响应，广播"我的IP是网关的IP，我的MAC是攻击者的MAC"。其他主机会更新ARP缓存，将所有发往网关的数据发送到攻击者主机上，实现中间人攻击（MITM）。防范措施：静态ARP表、DAI（Dynamic ARP Inspection）。
考试要点：ARP只在同一局域网内工作。如果目的主机在不同网络，主机通过ARP获取默认网关的MAC地址，将数据交给网关路由器转发。

4.4.2 IP分片（必考！）

为什么需要分片？

每种物理网络都有最大传输单元（MTU）。以太网的MTU是1500字节。如果IP数据报大于MTU，路由器必须将其拆分成多个较小的分片发送，到达目的主机后再重组。

分片规则

  • 分片只拆分数据部分，首部复制到每个分片中
  • 各分片的标识字段相同（属于同一原始数据报）
  • 非最后分片MF=1，最后分片MF=0
  • 片偏移以8字节为单位，即偏移值 = 字节偏移 / 8
  • 每个分片的数据长度必须是8的倍数（最后一片除外）

完整例题：IP分片计算

题目：一个IP数据报总长度为4000字节（首部20字节，数据3980字节），链路的MTU=1500字节。请计算各分片的标识、MF标志、片偏移和总长度。

解题思路：

第一步：确定每片能携带的最大数据量
MTU = 1500字节
每片首部 = 20字节
每片最多携带数据 = 1500 - 20 = 1480字节

第二步：检查1480是否是8的倍数
1480 / 8 = 185 ✓（整除，符合要求）

第三步：计算需要多少片
总数据 = 3980字节
片数 = ceil(3980 / 1480) = ceil(2.689) = 3片

第四步：计算每片的数据长度
前两片：各1480字节
最后一片：3980 - 1480 × 2 = 1020字节

第五步：计算每片的片偏移
片1的偏移：0 / 8 = 0
片2的偏移：1480 / 8 = 185
片3的偏移：(1480 + 1480) / 8 = 2960 / 8 = 370
IP分片结果
分片标识MF片偏移总长度数据长度
分片1123451（还有后续）015001480
分片2123451（还有后续）18515001480
分片3123450（最后一片）37010401020
 
验证：
原始数据 = 1480 + 1480 + 1020 = 3980 ✓
最后一片的数据1020不是8的倍数？没问题——最后一片无此要求

4.5 子网划分（超级重点）

为什么需要子网划分？

分类IP地址有个严重问题：地址浪费。一个B类网络有65534个主机号，但大多数组织用不了这么多。同时，一个C类网络只有254个主机号，对稍大一点的组织又不够。

子网划分的思路是：从主机号中借用若干位作为子网号，将一个大网络划分为多个小网络。这样既能减少地址浪费，又能缩小广播域、方便管理。

子网划分核心概念

  • 子网掩码：32位二进制数，网络号和子网号部分全1，主机号部分全0
  • 用 IP地址 AND 子网掩码 即可得到子网地址
  • 子网地址 = 网络号 + 子网号 + 主机号全0

完整例题：B类地址子网划分

题目：某组织拥有B类地址150.10.0.0，需要划分4个子网。求子网掩码和每个子网的地址范围。

解题步骤：

第一步：确定需要借多少位
需要4个子网 → 2² = 4，需要借2位

第二步：计算子网掩码
B类默认掩码 = 255.255.0.0（前16位为1）
借2位后，网络位 = 16 + 2 = 18位
子网掩码 = 255.255.192.0

为什么是192？因为：
第3字节的前2位是子网号，对应二进制 11000000 = 192

第三步：列出4个子网（变化的是第3字节的前2位）
子网1：150.10.00 000000.0 → 150.10.0.0
子网2：150.10.01 000000.0 → 150.10.64.0
子网3：150.10.10 000000.0 → 150.10.128.0
子网4：150.10.11 000000.0 → 150.10.192.0

第四步：计算每个子网的主机地址范围
每个子网的主机位 = 16 - 2 = 14位（主机数 = 2¹⁴ - 2 = 16382）

子网1：150.10.0.1 ~ 150.10.63.254
子网2：150.10.64.1 ~ 150.10.127.254
子网3：150.10.128.1 ~ 150.10.191.254
子网4：150.10.192.1 ~ 150.10.255.254

注意：全0主机号和全1主机号（广播地址）不能分配给主机

4.6 CIDR（超级重点）

为什么需要CIDR？

分类IP地址不仅浪费，而且路由表膨胀——每个A/B/C类网络都需要一条路由条目。CIDR（无分类域间路由选择）解决了这两个问题：

  • 消除分类：不再有A/B/C类的概念，IP地址 = 网络前缀 + 主机号
  • 斜线记法：a.b.c.d/n，n是网络前缀的位数
  • 路由聚合：多条路由条目可以合并为一条，大幅缩小路由表

CIDR地址块计算

CIDR核心概念

  • 一个CIDR地址块由最小地址和最大地址界定
  • 地址数量 = 2(32 - n)
  • 最小地址：网络前缀不变，主机号全0
  • 最大地址：网络前缀不变，主机号全1

例题：CIDR地址块计算

题目：已知地址128.14.35.7/20，求该地址块的最小地址、最大地址和地址数量。

解题过程：

第一步：写出32位二进制
128.14.35.7 = 10000000.00001110.00100011.00000111

第二步：前20位是网络前缀（下划线部分）
            10000000.00001110.0010 0011.00000111
            └──────20位──────┘└──12位主机号──┘

第三步：最小地址 = 前缀不变 + 主机号全0
            10000000.00001110.0010 0000.00000000
            = 128.14.32.0

第四步：最大地址 = 前缀不变 + 主机号全1
            10000000.00001110.0010 1111.11111111
            = 128.14.47.255

第五步：地址数量 = 2¹² = 4096个地址

注意：最小地址128.14.32.0不是从128.14.35.0开始的！
因为20位前缀对应的第3字节是前4位0010 = 32

例题：CIDR地址分配

题目：ISP拥有地址块206.0.64.0/18，需要分配给4个组——A组500个地址、B组250个地址、C组120个地址、D组60个地址。请按从大到小分配，给出每组的前缀长度和地址范围。

解题步骤：

第一步：确定各组的地址需求（2的幂）
组   需求    实际分配    所需主机位    前缀长度
A   500      512(2⁹)    9位          32-9=23
B   250      256(2⁸)    8位          32-8=24
C   120      128(2⁷)    7位          32-7=25
D   60        64(2⁶)    6位          32-6=26

第二步：从大到小分配（地址块从206.0.64.0/18开始）

ISP的地址块范围：
206.0.64.0/18 → 最小206.0.64.0，最大206.0.127.255
（第3字节64=01000000，前2位是前缀的一部分）

第三步：分配A组（500个，/23）
206.0.64.0/23 → 206.0.64.0 ~ 206.0.65.255（512个地址）

第四步：分配B组（250个，/24）
接续地址：206.0.66.0/24 → 206.0.66.0 ~ 206.0.66.255（256个地址）

第五步：分配C组（120个，/25）
接续地址：206.0.67.0/25 → 206.0.67.0 ~ 206.0.67.127（128个地址）

汇总：
A组：206.0.64.0/23   (206.0.64.0 ~ 206.0.65.255)
B组：206.0.66.0/24   (206.0.66.0 ~ 206.0.66.255)
C组：206.0.67.0/25   (206.0.67.0 ~ 206.0.67.127)
D组：206.0.67.128/26  (206.0.67.128 ~ 206.0.67.191)

最长前缀匹配

路由器在转发分组时，可能同时匹配多个路由条目。CIDR的策略是选择前缀最长的那一条——前缀越长，地址范围越小，匹配越精确。

为什么是最长前缀？ 假设有两条路由条目：
206.0.64.0/18（范围大）和 206.0.66.0/24（范围小）
目标地址206.0.66.1同时匹配这两条，但/24更精确——选择最精确的，而不是最宽泛的。

4.7 ICMP协议

为什么需要ICMP？

IP协议只负责尽力送达，不提供差错报告机制。当路由器发现无法转发某个分组（如TTL超时、目的不可达），它需要通知发送方——这就是ICMP（互联网控制报文协议）的作用。

ICMP报文类型

常见ICMP报文
类型类别说明
0询问回送应答（Echo Reply）
3差错目的不可达（Destination Unreachable）
8询问回送请求（Echo Request）
11差错超时（Time Exceeded，TTL减到0）

Ping和Traceroute原理

1

Ping

发送ICMP回送请求（Type=8），目的主机回复回送应答（Type=0）。通过测量请求-应答的往返时间（RTT），判断目标是否可达及网络延迟。这是最常用的网络诊断工具。

2

Traceroute

利用TTL字段和ICMP超时报文：依次发送TTL=1,2,3,...的UDP数据报，每个路由器转发时TTL减1，减到0时丢弃并向源主机发送ICMP超时报文。这样就能逐一发现路径上的所有路由器。

4.8 路由协议（必考对比）

为什么需要路由协议？

互联网由成千上万个自治系统（AS）组成。路由器之间需要交换网络可达信息，才能知道某个IP地址往哪个方向转发。路由协议就是路由器之间通信的\"语言\"。按作用范围分为两大类：

  • IGP（内部网关协议）：AS内部使用，包括RIP和OSPF
  • EGP（外部网关协议）：AS之间使用，目前只有BGP

RIP（路由信息协议）

RIP要点

  • 算法：距离向量（Distance Vector），基于Bellman-Ford算法
  • 度量（Metric）：跳数（Hop Count），每经过一个路由器跳数+1
  • 最大跳数：15跳（16表示不可达）→ RIP只适合小型网络
  • 更新方式：每30秒广播完整路由表到邻居
  • 收敛慢：路由变化后收敛时间长，可能出现\"坏消息传得慢\"问题

OSPF（开放最短路径优先）

OSPF要点

  • 算法：链路状态（Link State），基于Dijkstra最短路径算法
  • 度量（Metric）：带宽相关（默认=10⁸/带宽bps），更精确
  • 更新方式：只在路由变化时发送链路状态通告（LSA），不是周期性广播
  • 收敛快：路由变化后能迅速收敛
  • 区域划分：支持将AS划分为多个区域，减少LSA传播范围
  • 5种报文：Hello、DD（数据库描述）、LSR（链路状态请求）、LSU（链路状态更新）、LSAck（链路状态确认）
  • 适合大型网络

RIP vs OSPF 详细对比

对比项RIPOSPF
算法类型距离向量（Distance Vector）链路状态（Link State）
核心算法Bellman-FordDijkstra（SPF）
度量标准跳数（Hop Count）带宽/开销（Cost）
最大度量15跳（16=不可达）无上限
更新方式每30秒广播完整路由表路由变化时发送LSA（增量更新）
收敛速度慢（可能数分钟）快（秒级）
适用范围小型网络大型网络
层次结构无（扁平结构）支持区域划分（Backbone Area + 普通区域）
路由环路有（需要毒性逆转等技术防环）无（每个路由器有完整拓扑图）
传输协议UDP（端口520）IP直传（协议号89）
报文类型2种（请求/响应）5种（Hello/DD/LSR/LSU/LSAck）

BGP（边界网关协议）

BGP要点

  • 位置：不同AS之间的路由协议（EGP）
  • 算法：路径向量（Path Vector）
  • 核心思想：BGP发言人交换的不仅是可达性信息，还包含完整的AS路径（如 AS1-AS2-AS5-AS7），用于避免环路
  • 4种报文：OPEN（建立连接）、UPDATE（更新路由）、NOTIFICATION（差错通知）、KEEPALIVE（保活）
  • 度量：不是简单的跳数或带宽，而是基于多种策略（经济、政治、性能等）

4.9 IPv6

为什么需要IPv6？

IPv4地址只有32位，最多43亿个地址。虽然通过NAT技术缓解了地址短缺，但这只是权宜之计。IPv6将地址扩展到128位，地址数量是IPv4的2⁹⁶倍——地球上每粒沙子都能分配一个IP地址。

IPv6地址特点

  • 长度：128位，是IPv4的4倍
  • 表示法：冒号十六进制，如 2001:0db8:85a3::8a2e:0370:7334
  • 简化规则：每组前导0可以省略，连续全0组可以用 :: 代替（只能使用一次）

IPv4 vs IPv6 对比

对比项IPv4IPv6
地址长度32位128位
表示法点分十进制冒号十六进制
首部长度20~60字节（可变）40字节（固定）
分片由路由器分片由源主机分片（路径MTU发现）
校验和有（首部校验和）无（由上层或链路层保证）
ARP使用ARP解析MAC使用NDP（邻居发现协议）
IPsec可选内建支持
地址配置DHCP（通常手动或DHCP）无状态地址自动配置（SLAAC）

过渡技术

IPv4到IPv6不能一蹴而就，需要过渡技术：

  • 双协议栈（Dual Stack）：设备同时运行IPv4和IPv6协议栈
  • 隧道技术（Tunneling）：将IPv6数据报封装在IPv4数据报中传输
  • NAT64/DNS64：帮助仅IPv6的客户端访问IPv4服务器

4.10 VPN与NAT

为什么需要VPN和NAT？

私有IP地址（10.x.x.x、172.16.x.x、192.168.x.x）在Internet上不能路由。但大多数家庭和企业使用的都是私有地址。他们上网时需要通过NAT将私有地址转换为公网地址。

同时，企业的不同分支机构（都在内网中）之间需要安全通信——VPN在公共互联网上建立一条加密隧道，让分布在不同地点的内网如同在一个局域网中。

NAT的工作原理

NAT（网络地址转换）

NAT将内部私有IP地址转换为公网IP地址，通常由路由器/NAT网关完成。其核心是维护一张NAT转换表，记录内部地址和外部地址的映射关系。

NAT工作流程（4步）：

1

内部主机发送数据

主机192.168.1.5发送一个数据报到Internet，源IP=192.168.1.5，源端口=3345。

2

NAT网关替换地址

NAT网关将源IP替换为自己的公网IP（如210.25.1.1），源端口也替换为一个新的端口号（如5001），并在NAT表中记录映射：192.168.1.5:3345 ↔ 210.25.1.1:5001。

3

公网传输和响应

数据报在Internet上传输，目的服务器回复的响应数据报发往210.25.1.1:5001。

4

NAT网关还原地址

NAT网关收到响应，查NAT表找到对应内部地址192.168.1.5:3345，将目的IP和端口替换回去，转发给内部主机。

NAPT（网络地址端口转换）

NAPT是NAT的扩展——通过端口号来区分不同内部主机的连接。多个内部主机可以共享同一个公网IP，因为端口号不同。你家的路由器就是NAPT！全家人的手机、电脑共享一个公网IP上网。

简答题

题7：ARP跨网段问题

题目：主机A（IP=192.168.1.2/24）想给主机B（IP=10.0.0.2/8）发送数据。主机A会如何获取主机B的MAC地址？请详细说明过程。

解答：

主机A通过子网掩码判断：192.168.1.2 AND 255.255.255.0 = 192.168.1.0，10.0.0.2 AND 255.255.255.0 = 10.0.0.0，两者不在同一网络。

因此主机A：

  1. 查路由表找到默认网关（假设为192.168.1.1）
  2. 通过ARP获取默认网关192.168.1.1的MAC地址
  3. 将数据帧的目的MAC设为网关的MAC，目的IP仍设为主机B的IP（10.0.0.2）
  4. 将帧发送给网关，由网关路由器转发

核心：ARP只在同一局域网内工作。跨网络通信时，ARP获取的是网关的MAC地址，而不是目的主机的MAC地址。

题8：RIP路由更新

题目：在RIP协议中，为什么跳数限制为15？"坏消息传得慢"是什么意思？

解答：

为什么限制15？RIP使用跳数作为度量，16表示不可达。限制15是为了防止路由环路——如果跳数无上限，路由更新可能在环路中无限循环。

"坏消息传得慢"：当某个网络不可达时（断网），好消息（该网络可达）传播很快，但坏消息（该网络不可达）传播很慢。因为路由器可能从邻居那里学习到已经失效的路由（邻居还没更新自己的路由表），导致路由环路。解决方法包括：毒性逆转（将该路由跳数设为16并广播）、水平分割（不从收到路由的接口再发回去）。
