武器进阶：四职业武器实现

真正决定武器手感的，不只是伤害数值，而是职业类型、动作节奏与弹幕逻辑的配合。

本章重点 从这一章开始，你写的就不再只是“能挥一下的物品”，而是开始进入 Terraria 战斗系统的核心：近战、远程、魔法、召唤四条路线。

武器的职业归属是怎么决定的

在现代 tModLoader 中，最关键的入口之一就是 Item.DamageType。它决定这把武器会吃哪些加成、归属于哪个职业体系、受到哪些前缀和装备效果影响。

DamageClass

DamageClass 是武器职业系统的核心抽象。常见的有 Melee、Ranged、Magic、Summon，此外还有 SummonMeleeSpeed 等更细分类型。

Item.damage = 24;
Item.DamageType = DamageClass.Melee;

这两行配在一起，才算真正告诉游戏：“这是一个近战武器，它的基础伤害是 24。”如果只写伤害不写类型，很多职业相关加成就会不对。

近战武器：最直观，但也最容易忽略手感细节

近战武器通常是玩家最先接触的类型，所以很多模组作者也喜欢从这里开始练。它的基础配置不难，但真正做出“顺手”的武器，要看攻速、挥动动作、击退和是否带弹幕。

using Terraria;
using Terraria.ID;
using Terraria.ModLoader;

namespace ExampleMod.Content.Items.Weapons;

public class EmberBlade : ModItem
{
    public override void SetDefaults()
    {
        Item.width = 40;
        Item.height = 40;
        Item.damage = 28;
        Item.DamageType = DamageClass.Melee;
        Item.useTime = 20;
        Item.useAnimation = 20;
        Item.useStyle = ItemUseStyleID.Swing;
        Item.knockBack = 6f;
        Item.UseSound = SoundID.Item1;
        Item.autoReuse = true;
        Item.value = Item.buyPrice(gold: 1);
        Item.rare = ItemRarityID.Green;
    }
}

useTime 和 useAnimation 到底差在哪

useTime 是“多长时间能再次触发一次使用”，useAnimation 是“整个动作播放多久”。它们可以相同，也可以不同。相同的时候更直观；不同的时候，武器手感会更微妙。

组合效果适合场景
useTime = useAnimation动作与判定节奏统一大多数基础武器
useTime < useAnimation动作持续更长，但判定更密连击感更强的武器
useTime > useAnimation通常手感奇怪一般不建议新手这么写
注意 武器“感觉不对”时，别先怀疑贴图和伤害，先回来看 useTime、useAnimation、useStyle 这三个字段的搭配。

投射型近战武器

不是所有近战都必须靠近敌人碰撞。你可以保留近战伤害类型，同时让武器发射弹幕，形成“挥剑时带剑气”的效果。

Item.noMelee = true;
Item.shoot = ModContent.ProjectileType<Projectiles.EmberWave>();
Item.shootSpeed = 10f;
Item.useStyle = ItemUseStyleID.Swing;
Item.DamageType = DamageClass.Melee;

这里的关键点是 noMelee = true。它关闭本体挥动碰撞伤害，只保留弹幕伤害。否则你可能会出现“挥砍一次打两段”的情况。

远程武器：重点在 ammo 与 shoot

远程武器的本质，通常不是物品本身直接打人，而是由它生成或调用某个弹幕，并且常常需要消耗弹药。弓、枪、发射器的差别，很多时候都体现在使用动作、音效和弹药来源上。

public class FrostRifle : ModItem
{
    public override void SetDefaults()
    {
        Item.width = 48;
        Item.height = 22;
        Item.damage = 18;
        Item.DamageType = DamageClass.Ranged;
        Item.useTime = 14;
        Item.useAnimation = 14;
        Item.useStyle = ItemUseStyleID.Shoot;
        Item.noMelee = true;
        Item.knockBack = 2f;
        Item.value = Item.buyPrice(gold: 1);
        Item.rare = ItemRarityID.Orange;
        Item.UseSound = SoundID.Item11;
        Item.autoReuse = true;
        Item.shoot = ProjectileID.Bullet;
        Item.useAmmo = AmmoID.Bullet;
        Item.shootSpeed = 14f;
    }
}

对于大多数枪械而言，你要看四个字段：shoot、useAmmo、shootSpeed、useStyle。它们决定“发什么”“吃什么弹药”“多快打出去”“使用时角色做什么动作”。

点击展开：pickAmmo 应该怎么理解

玩家背包里可能有多种可用弹药。tModLoader/原版逻辑会帮助你从可用弹药中挑出实际消耗对象，并据此决定弹幕种类和修正值。你不一定每次都要手动写这一层，但你要知道远程武器并不只是“shoot 一个东西”那么简单。

魔法武器：mana 是职业边界的重要信号

魔法武器通常也依赖弹幕，但和远程武器不同，它不消耗弹药，而消耗法力值。只要法力消耗、伤害类型、使用动作配对正确，玩家就会自然把它识别成法师武器。

public class CrystalBoltStaff : ModItem
{
    public override void SetDefaults()
    {
        Item.width = 40;
        Item.height = 40;
        Item.damage = 32;
        Item.DamageType = DamageClass.Magic;
        Item.mana = 8;
        Item.useTime = 24;
        Item.useAnimation = 24;
        Item.useStyle = ItemUseStyleID.Shoot;
        Item.noMelee = true;
        Item.knockBack = 3f;
        Item.value = Item.buyPrice(gold: 2);
        Item.rare = ItemRarityID.LightRed;
        Item.UseSound = SoundID.Item20;
        Item.shoot = ModContent.ProjectileType<Projectiles.CrystalBolt>();
        Item.shootSpeed = 12f;
    }
}

这里最重要的新字段是 mana。法力消耗会直接影响战斗节奏，也决定装备与饰品的联动价值。你以后做法师体系时，平衡常常不是“伤害给多少”，而是“每点法力值能换来多大收益”。

召唤武器：真正的重点在弹幕那一边

召唤武器的物品端配置并不复杂，真正复杂的是它生成的 Minion 弹幕如何跟随玩家、如何寻找敌人、如何占用召唤栏位。也就是说，召唤武器往往是“Item + Projectile 的组合工程”。

public class TinyFlameBell : ModItem
{
    public override void SetDefaults()
    {
        Item.width = 28;
        Item.height = 30;
        Item.damage = 16;
        Item.DamageType = DamageClass.Summon;
        Item.mana = 10;
        Item.useTime = 30;
        Item.useAnimation = 30;
        Item.useStyle = ItemUseStyleID.Swing;
        Item.noMelee = true;
        Item.knockBack = 2f;
        Item.UseSound = SoundID.Item44;
        Item.shoot = ModContent.ProjectileType<Projectiles.TinyFlameMinion>();
        Item.buffType = ModContent.BuffType<Buffs.TinyFlameMinionBuff>();
        Item.buffTime = 2;
        Item.noUseGraphic = false;
        Item.value = Item.buyPrice(gold: 1);
        Item.rare = ItemRarityID.Blue;
    }

    public override bool? UseItem(Player player)
    {
        player.AddBuff(Item.buffType, 2);
        return true;
    }
}

你会发现这类武器的“攻击行为”基本不在 Item 里写完，而是在对应的 ModProjectile 里真正实现。实际项目里，召唤杖通常还会配一个专属 Buff，用来维持召唤物存在，并在 UseItem() 里给玩家补上这个 Buff。也就是说，写召唤武器时，要从一开始就接受一个事实：物品只是召唤入口，不是完整战斗逻辑本体。

设计建议 召唤武器特别适合做“成长感”。哪怕武器本身只是一次召唤动作，只要 minion 的跟随、索敌、攻击节奏做得好，玩家就会觉得它很有生命力。

击退系统不是可有可无的配角

Item.knockBack 决定命中时的击退强度，但实际效果还会和敌人的击退抗性一起计算。高击退不一定等于高强度，因为某些怪物天生抗击退，而某些快攻武器即使伤害低，靠击退也能打出优秀生存空间。

Item.knockBack = 1.5f; // 轻击退
Item.knockBack = 4.5f; // 中等
Item.knockBack = 7f;   // 明显击退

做武器平衡时，击退和伤害要一起看。比如一把伤害中等但能稳定把敌人推开的枪，实际战斗压力可能比纸面伤害更高的短射程法杖还低。

做一把“像样”的武器，通常要经历这几个步骤

1

先确定职业

别一上来堆字段，先决定它属于近战、远程、魔法还是召唤。

2

确定输出媒介

是本体碰撞、射出弹幕，还是召唤 minion。媒介决定后面的绝大部分配置。

3

调手感字段

优先调 useTime、useAnimation、shootSpeed、knockBack。

4

最后再调伤害和价格

因为手感稳定前，数值平衡意义不大。

常见错误

  • 写了 damage 却忘了 DamageType。
  • 远程武器配置了 shoot 却没写 useAmmo，导致定位不清。
  • 投射型近战忘了 noMelee = true，结果本体和弹幕双重伤害。
  • 召唤武器以为 Item 写完就结束，忽略了 Minion 的 Projectile 逻辑。
别急着追求“花活” 初学时最容易掉进的坑，是一上来就想做会分裂、会转向、会多阶段切换的复杂武器。更稳的做法是：先做一把职业归属清晰、节奏稳定、参数合理的基础武器。

本章小结

现在你已经能从职业视角去理解武器，而不是只把它当成“带伤害的物品”。近战强调动作和击退，远程强调弹药与射击逻辑，魔法强调法力成本，召唤则更依赖后端的 minion 行为。
