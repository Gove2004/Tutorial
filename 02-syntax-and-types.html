02 - 基础语法与类型系统

Lua 的类型系统极其精简——只有 8 种类型。但精简不等于简单：nil 的多重语义、number 的整数/浮点双表示、table 的万能性、以及动态类型带来的灵活性，都值得深入理解。

变量与赋值

Lua 的变量默认为全局变量——这是它与大多数语言最大的不同。局部变量必须显式地用 local 声明。

-- 全局变量（默认）
score = 100
player_name = "Gove"

-- 局部变量（推荐）
local level = 5
local health, mana = 100, 50
性能陷阱：全局变量访问比局部变量慢一个数量级。全局变量储存在 _G 表中，每次访问都是一次哈希查找；局部变量在寄存器中，几乎零开销。在生产代码中，永远用 local。

多重赋值

Lua 支持多变量同时赋值，右侧先求值，再从左到右赋给左侧：

-- 经典交换，无需临时变量
local a, b = 10, 20
a, b = b, a          -- a=20, b=10

-- 多返回值接收
local x, y = 1, 2, 3  -- x=1, y=2, 3 被丢弃
local p, q = 5        -- p=5, q=nil（不足时补 nil）

Lua 的 8 种类型

使用 type() 函数可以查看任何值的类型。返回值是一个字符串。

类型type() 返回值说明
nil"nil"空值、未声明、不存在——唯一的「假」值之一
boolean"boolean"true/false。注意：0 和空字符串都是 true！
number"number"整数（64位有符号）和浮点（64位双精度）的双表示
string"string"不可变字节序列，支持单引号、双引号、长括号三种写法
table"table"唯一的容器类型，同时承担数组、字典、对象
function"function"一等公民，可赋值、传参、返回、闭包捕获
userdata"userdata"C 层对象在 Lua 中的代理，分 full userdata 和 light userdata
thread"thread"协程对象
 
print(type(nil))         --> nil
print(type(true))        --> boolean
print(type(42))          --> number
print(type(3.14))        --> number
print(type("hello"))     --> string
print(type({}))          --> table
print(type(print))       --> function

nil：不只是空

nil 在 Lua 中有多重语义：

  • 空值：表示「没有值」
  • 未定义：访问未赋值的变量 → nil
  • 删除：将 table 字段设为 nil = 删除该字段
  • 不存在：访问 table 中不存在的 key → nil
  • 条件假：nil 和 false 是仅有的两个「假」值
local t = {name = "Gove", age = 21}
t.age = nil            -- 删除 age 字段
print(t.age)           --> nil
print(#t)              -- 不适用，详见后续章节

-- nil 的假值行为
if nil then print("yes") else print("no") end   --> no
与 C/C++/Python 的关键差异：Lua 中 0 和空字符串 "" 都是 truthy。只有 nil 和 false 为假。即 if 0 then 为 true！这在从 C 转 Lua 时是最容易写错的 bug。

number：双表示的数值类型

Lua 5.3+ 的 number 类型内部有两种表示：integer（64位有符号）和 float（64位双精度），但对外统一显示为 "number"。Lua 在运算时自动在两者间转换。

local i = 42             -- 整数表示
local f = 3.14           -- 浮点表示

-- 整数和浮点可以互相转换
print(math.type(42))     --> integer
print(math.type(42.0))   --> float（显式写了小数点）

-- 算术运算符自动处理类型
print(42 + 3.14)         --> 45.14（整数被提升为浮点）
print(10 // 3)           --> 3（整除，返回整数）
print(10.0 // 3)         --> 3.0（操作数有浮点则返回浮点）

-- 位运算符仅对整数有效
print(1 << 3)            --> 8
-- print(1.5 << 3)      -- 运行时错误！位运算需要整数
历史注记：Lua 5.2 及之前所有数字都是浮点（双精度）。5.3 引入整数子类型是一个重大变化，解决了游戏开发中位运算和精确计数的需求。LuaJIT 2.x 也是整数+浮点双表示（基于 5.1 但自己实现了）。

string：不可变的字节序列

Lua 的字符串是不可变的——每次拼接都产生新字符串。它不是「字符」序列而是「字节」序列，天然支持任意二进制数据。

-- 三种字面量写法
local s1 = '单引号'
local s2 = "双引号"
local s3 = [[
长括号：包含 "引号" 和
换行都原样保留
无需转义
]]

-- 长括号可以有等号层数，防止内容本身含 ]]
local sql = [==[
SELECT * FROM users WHERE name = "O'Brien";
-- 内容含 ]] 也不怕
]==]

-- 字符串拼接用 ..
local name = "Lua"
print("Hello, " .. name .. "!")  --> Hello, Lua!

-- 长度用 #
print(#"hello")                   --> 5

-- 字符串是不可变的
local s = "hello"
s:upper()          -- 返回 "HELLO"，s 不变
print(s)           --> hello
推荐写法不推荐写法
 
-- 长括号，可读性好
local json = [[
{
  "name": "Gove",
  "age": 21
}
]]
 
-- 引号 + 转义，混乱
local json = "{\n  \"name\": \"Gove\",\n  \"age\": 21\n}"

运算符

Lua 的运算符设计极简，没有 ++ 和 --，没有三元运算符，没有复合赋值。这看起来简陋，其实是 Lua「少即是多」哲学的延续。

类别运算符示例
算术+ - * / // ^ %2^10 → 1024.0
关系== ~= < > <= >=不等号是 ~= 而非 !=
逻辑and or notx = a and b or c
拼接.."a" .. "b" → "ab"
长度##"hello" → 5
位运算 (5.3+)& | ~ << >> //0xFF & 0x0F → 15
逻辑运算符的短路值：a and b 若 a 为假则返回 a，否则返回 b。a or b 若 a 为真则返回 a，否则返回 b。这不是返回 boolean！这是 Lua 实现「默认值」和「三元运算」的机制。 
-- 用 or 实现默认值
local name = input or "anonymous"

-- 用 and/or 模拟三元运算符：条件 and 真值 or 假值
local status = (score > 60) and "pass" or "fail"

-- ⚠ 注意：真值不能是 nil/false，否则逻辑被打破
local x = false or "default"   --> "default"（期望 false 却被吞了）

控制流

if-then-elseif-else

local score = 85

if score >= 90 then
    print("优秀")
elseif score >= 80 then    -- 注意：是 elseif 不是 else if！
    print("良好")
elseif score >= 60 then
    print("及格")
else
    print("不及格")
end

while 循环

local i = 1
while i <= 5 do
    print(i)
    i = i + 1       -- Lua 没有 i++
end

repeat-until（先执行后判断）

local i = 1
repeat
    print(i)
    i = i + 1
until i > 5              -- 条件为真时退出（和 while 相反）
repeat-until 的独特性：循环体内的局部变量在 until 条件中也可见——和 while 不同。这是 Lua 特意设计的，很适合「读-处理-判断」模式。

for 循环：数值 for

-- for 初始值, 终止值, 步长（可选，默认 1）
for i = 1, 5 do
    print(i)            -- 1, 2, 3, 4, 5
end

for i = 10, 1, -2 do
    print(i)            -- 10, 8, 6, 4, 2
end

-- 关键行为：三个表达式只在循环开始时求值一次！
-- 循环内修改控制变量是无效的
local limit = 5
for i = 1, limit do
    limit = 10          -- 不影响循环次数，仍是 5 次
    print(i)
end

for 循环：泛型 for（迭代器）

-- ipairs：按序迭代数组部分
local arr = {"a", "b", "c"}
for i, v in ipairs(arr) do
    print(i, v)          -- 1 a, 2 b, 3 c
end

-- pairs：迭代所有键值对
local t = {name = "Gove", age = 21}
for k, v in pairs(t) do
    print(k, v)          -- name Gove, age 21（顺序不保证）
end

-- 迭代字符串中的单词
for word in string.gmatch("hello world lua", "%S+") do
    print(word)          -- hello, world, lua
end

变长参数

函数可以接收不定数量的参数，用 ... 表示：

local function sum(...)
    local total = 0
    for _, v in ipairs({...}) do   -- 将变长参数打包成 table
        total = total + v
    end
    return total
end

print(sum(1, 2, 3, 4, 5))          --> 15

-- 传参给另一个函数
local function log_and_call(f, ...)
    print("calling function")
    return f(...)                   -- 透传参数
end

-- 固定参数 + 变长参数
local function greet(greeting, ...)
    local names = {...}
    for _, name in ipairs(names) do
        print(greeting .. ", " .. name)
    end
end

greet("Hello", "Gove", "Alice", "Bob")
-- Hello, Gove
-- Hello, Alice
-- Hello, Bob
table.pack 更好：{...} 打包时不会保留 nil 参数（无法区分「传了 nil」和「没传」）。table.pack(...)（5.2+）返回的 table 有 .n 字段记录实际参数个数，更安全。

local vs global 的性能真相

这不是「建议用 local」——这是 Lua 性能的第一课。看看为什么：

-- 全局变量访问：_G["x"] → 哈希查找
x = 100
for i = 1, 10000000 do
    x = x + 1              -- 每次循环做两次哈希查找
end

-- 局部变量访问：寄存器
local y = 100
for i = 1, 10000000 do
    y = y + 1              -- 直接寄存器操作
end
-- 局部版本的循环快 5-10 倍

局部变量除了快，还限制作用域，防止全局命名空间污染。在一个文件中 x = 1，所有被 require 的文件都能看到它——这是灾难的源头。

黄金法则：所有变量都声明为 local。如果你需要全局共享，显式写入 _G.my_var = value 或返回一个 table 给 require 的调用者。隐式全局是 bug 的温床。

练习与思考

1

基础练习

写出 Lua 中仅有的两个「假」值。然后测试 if "" then 和 if 0 then 的结果，确认你的理解。

2

编码实践

用 and/or 实现一个 clamp(x, min, max) 函数——将数值限制在区间内。不要用 if 语句。

3

对比分析

Lua 的 for i = 1, n 在循环开始时就确定了循环次数，中途修改变量 n 不会影响循环。Python 的 for i in range(n) 同理，但 C 的 for(i=0;i 每次都重新求值。这三种设计各有什么优缺点？

本章小结

  • Lua 只有 8 种类型，type() 函数返回类型名
  • nil 是多面手——空值、未定义、删除、假值。0 和 "" 都是真值，这与 C/Python 完全不同
  • number 内部是整数和浮点双表示（5.3+），算术运算自动转换
  • 字符串不可变，拼接用 ..，长括号 [[ ]] 最易读
  • 逻辑运算符 and/or 返回短路值（非 boolean），可用于默认值和三元替代
  • 永远用 local 声明变量——既为性能也为正确性
