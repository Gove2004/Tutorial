物品基础

从 ModItem 开始，学会往游戏里稳定地加入“能拿、能看、能用”的内容。

为什么先学物品 物品是 tModLoader 里最适合入门的内容类型。它既能训练你理解内容注册，也能让你快速看到游戏内反馈。

ModItem 的定位

在 tModLoader 里，几乎所有可进入背包、可掉落、可拾取、可使用的东西，都可以围绕 ModItem 来理解。你写的类不是“游戏中那一把具体的剑实例”，而是“这类物品的定义模板”。游戏在运行时会根据它生成实际物品实例。

ModItem

继承自 ModItem 的类，用来定义一个自定义物品的属性、使用方式、提示文本、合成与交互逻辑。

一个最小可用物品

using Terraria;
using Terraria.ID;
using Terraria.ModLoader;

namespace ExampleMod.Content.Items;

public class CopperPracticeSword : ModItem
{
    public override void SetStaticDefaults()
    {
        Item.ResearchUnlockCount = 1;
    }

    public override void SetDefaults()
    {
        Item.width = 40;
        Item.height = 40;
        Item.damage = 14;
        Item.DamageType = DamageClass.Melee;
        Item.useTime = 20;
        Item.useAnimation = 20;
        Item.useStyle = ItemUseStyleID.Swing;
        Item.knockBack = 5f;
        Item.value = Item.buyPrice(silver: 35);
        Item.rare = ItemRarityID.White;
        Item.UseSound = SoundID.Item1;
        Item.autoReuse = true;
    }
}

这段代码已经足够让一把基础近战武器进入游戏。你现在最应该盯住的是 SetDefaults()：大部分基础属性都在这里定义。为了和后续章节形成连续练习路线，这一章建议你把它当成“第一个原型物品”，后面再逐步把它演化成带配方、带弹幕、带更强战斗表现的完整武器线。

SetDefaults 里最常见的字段

字段作用你要注意什么
width / height物品图标尺寸通常与贴图尺寸对应
damage基础伤害只对有伤害能力的物品有意义
DamageType伤害类型决定受哪些加成影响
useTime使用间隔越小越快
useAnimation动作总时长与 useTime 配合决定手感
useStyle使用动作类型挥砍、举起、射击等
rare稀有度决定物品名颜色
value价格建议用 Item.buyPrice
经验建议 刚开始不要一口气记住几十个字段。先把“尺寸、伤害、伤害类型、使用动作、攻速、价格、稀有度”这几个核心属性用熟，后面再自然扩展。

Tooltip 和静态数据

很多教程会用旧写法设置提示文本，但你在新版本思路里更应该把 SetStaticDefaults() 看成“静态展示信息的配置点”。研究解锁数量、名称、本地化文本、一些固定说明都更适合放在这里。

public override void SetStaticDefaults()
{
    Item.ResearchUnlockCount = 1;
}
点击展开：为什么这类东西不放进 SetDefaults

SetDefaults() 更偏向“每个实例都要有的运行属性”，而 SetStaticDefaults() 更像“这个物品类型的静态定义”。分清两者，后面写 Buff、NPC、弹幕都会更顺。

物品贴图怎么对应代码

tModLoader 的资源系统很强调命名与路径约定。最稳妥的方式，是让类名、文件名、资源路径保持一致。例如类名是 CopperPracticeSword，那贴图文件通常也叫 CopperPracticeSword.png；而像 StarOreItem 这种放置物块物品，则通常会在后续章节里和 StarOreTile 配对出现。

1

建立 Items 目录

代码通常放在 Content/Items/，贴图也尽量用相近的语义路径组织。

2

保持类名和贴图名一致

这样最不容易出错，排查资源问题也最直观。

3

常见尺寸先按 40x40 练手

不是硬性规则，但对初学做普通物品图标很友好。

注意 贴图不显示时，先查命名和路径，再查代码。很多“代码看起来没问题”的显示 bug，最后都是资源文件名不匹配。

物品不只有武器

当你说“我要做个新物品”时，别默认它一定是武器。tModLoader 里的物品大致可以先分成几类：武器、材料、消耗品、工具、可放置物块的物品、饰品。它们都能共用 ModItem 这套入口，但行为重点不同。

材料物品

using Terraria;
using Terraria.ID;
using Terraria.ModLoader;

namespace ExampleMod.Content.Items.Materials;

public class StarOreChunk : ModItem
{
    public override void SetDefaults()
    {
        Item.width = 26;
        Item.height = 24;
        Item.maxStack = 9999;
        Item.value = Item.buyPrice(silver: 2);
        Item.rare = ItemRarityID.Blue;
    }
}

材料的重点通常不是使用动作，而是堆叠数、价值、稀有度，以及后续参与哪些配方。

可放置物块的物品

using Terraria;
using Terraria.ID;
using Terraria.ModLoader;

namespace ExampleMod.Content.Items.Placeables;

public class StarOreItem : ModItem
{
    public override void SetDefaults()
    {
        Item.width = 20;
        Item.height = 20;
        Item.maxStack = 9999;
        Item.useTurn = true;
        Item.autoReuse = true;
        Item.useAnimation = 15;
        Item.useTime = 10;
        Item.useStyle = ItemUseStyleID.Swing;
        Item.consumable = true;
        Item.createTile = ModContent.TileType<Content.Tiles.StarOreTile>();
    }
}

这里最关键的字段是 consumable 和 createTile。它们告诉游戏：这个物品被使用后会消耗，并在世界中放置指定物块。

实战：一把练手剑、一个矿石材料、一个可放置方块

如果你只想快速建立完整闭环，最适合的组合就是这三种。它们分别训练你三个核心能力：战斗属性配置、基础物品配置、与世界物块建立关联。

练习路线 先做剑确认基础属性生效，再做矿石材料确认图标和堆叠逻辑，再做可放置方块物品，为下一章的 ModTile 铺路。

常见问题

为什么物品能加载但手感不对

多数情况下是 useTime、useAnimation、useStyle 三者搭配不合理。它们不会报错，但会让武器表现得“怪怪的”。

为什么物品有伤害却打不出效果

先查伤害类型、使用动作和是否真正属于近战/远程/魔法逻辑；再查是否需要弹幕支持。很多“武器没反应”其实不是 Item 写错，而是攻击方式依赖的后续系统还没接上。

本章小结

你现在应该已经掌握了最关键的物品思路：ModItem 负责定义物品类型，SetDefaults() 负责配置大部分运行属性，资源命名决定显示是否正常，不同物品只是在同一入口下强调不同字段。
