03 - 函数、闭包与尾调用

如果说 table 是 Lua 的数据核心，那函数就是行为核心。Lua 的函数是一等公民（first-class），加上闭包和 proper tail call 两大利器，让 Lua 拥有了远超其体量的表达能力。

函数是一等公民

一等公民

一个值是「一等公民」意味着它可以：赋给变量、作为参数传递、作为返回值、匿名存在、在运行时创建。Lua 的函数满足所有条件。

-- 函数是值，可以赋给变量
local greet = function(name)
    return "Hello, " .. name
end

-- 等价语法糖
local function greet(name)
    return "Hello, " .. name
end

-- 作为参数传递（高阶函数）
local function apply(func, value)
    return func(value)
end
print(apply(greet, "Gove"))  --> Hello, Gove

-- 作为返回值（函数工厂）
local function make_multiplier(n)
    return function(x)
        return x * n
    end
end
local double = make_multiplier(2)
print(double(5))              --> 10

多返回值

Lua 函数可以返回多个值——这不是返回一个 tuple/list，而是真的在栈上返回多个值。

-- 多返回值
local function stats(numbers)
    local total, count = 0, 0
    for _, v in ipairs(numbers) do
        total = total + v
        count = count + 1
    end
    return total, count, total / count
end

local sum, n, avg = stats({85, 92, 78, 95})
print(sum, n, avg)  --> 350  4  87.5

-- 多返回值只在调用位于表达式列表末尾时才全部保留
local function f() return 1, 2, 3 end

local a = f()               -- a = 1, 2 和 3 被丢弃
local a, b = f()            -- a = 1, b = 2, 3 被丢弃
local a, b, c = f()         -- a = 1, b = 2, c = 3
local a, b = f(), 99       -- a = 1, b = 99（f() 被截断为单值）

-- table 构造器中自动收集所有返回值
local t = {f()}            -- {1, 2, 3}

-- 函数调用时透传多返回值
print(f())                 --> 1  2  3
常见坑：t = {f()} 收全了 3 个值，但 t = {f(), 99} 只收到 {1, 99}——因为 f() 不在末尾位置，多返回值被截断为 1 个。

闭包：函数 + 环境

闭包（closure）是 Lua 最强大的特性之一：一个函数加上它捕获的所有外部局部变量（称为 upvalue）。

upvalue 的工作方式

-- upvalue 是共享的引用，不是拷贝
local function counter()
    local count = 0           -- 这个局部变量对所有返回的闭包共享
    return {
        increment = function()
            count = count + 1  -- 修改同一个 upvalue
            return count
        end,
        decrement = function()
            count = count - 1  -- 修改同一个 upvalue
            return count
        end,
        get = function()
            return count
        end,
    }
end

local c = counter()
c.increment(); c.increment()
print(c.get())            --> 2
c.decrement()
print(c.get())            --> 1

闭包陷阱：循环中的闭包

这是每个 Lua 学习者都会踩的坑：

错误写法正确写法
 
local funcs = {}
for i = 1, 5 do
    funcs[i] = function()
        return i    -- 都是 6！
    end
end
-- 所有闭包共享同一个 i
 
local funcs = {}
for i = 1, 5 do
    local j = i    -- 每次循环新变量
    funcs[i] = function()
        return j    -- 捕获的是不同的 j
    end
end
-- funcs[3]() → 3 ✓

原因：for 循环的控制变量 i 只有一个，每次迭代复用同一个变量（修改其值）。所有闭包指向同一个 i。解决方案是每次创建一个新的局部变量 j 作为快照。

闭包的实战用途

-- 1. 迭代器（自定义迭代协议）
local function range(from, to, step)
    step = step or 1
    local current = from - step
    return function()
        current = current + step
        if current <= to then
            return current
        end
    end
end

for i in range(1, 5) do
    print(i)    -- 1, 2, 3, 4, 5
end

-- 2. 装饰器模式（计时函数的执行时间）
local function timed(func)
    return function(...)
        local start = os.clock()
        local results = {func(...)}
        local elapsed = os.clock() - start
        print(string.format("耗时: %.4f 秒", elapsed))
        return table.unpack(results)
    end
end

local slow_sum = timed(sum)

-- 3. 回调注册（事件系统）
local function create_event_system()
    local handlers = {}
    return {
        on = function(event, callback)
            handlers[event] = handlers[event] or {}
            table.insert(handlers[event], callback)
        end,
        emit = function(event, ...)
            for _, cb in ipairs(handlers[event] or {}) do
                cb(...)
            end
        end,
    }
end

local events = create_event_system()
events.on("death", function(name) print(name .. " died") end)
events.emit("death", "Gove")   --> Gove died

Proper Tail Call（正确尾调用）

这是 Lua 独有的特性——不是优化，是语言规范保证的行为。

Proper Tail Call

当函数的最后一个动作是调用另一个函数（且调用后不需要做任何处理），Lua 会回收当前栈帧，让被调用函数直接复用。这意味着无限尾递归不会栈溢出，且不消耗额外内存。

-- 尾调用：return 后直接跟函数调用，不做任何其他操作
local function foo() return bar() end    -- ✓ 尾调用

-- 不是尾调用的情况
local function foo() return bar() + 1 end     -- ✗ 还要加 1
local function foo() return (bar()) end        -- ✗ 括号截断了
local function foo() local x = bar() end      -- ✗ 没有 return
local function foo() x = bar(); return x end  -- ✗ return 的不是函数调用

-- 经典的尾递归状态机，永不栈溢出
local function state_machine(state, data)
    if state == "done" then
        return data
    elseif state == "parse" then
        local result = parse(data)
        return state_machine("validate", result)  -- 尾调用
    elseif state == "validate" then
        local ok = validate(data)
        return state_machine("done", data)        -- 尾调用
    end
end
-- 即使跑 100000 次迭代也不会栈溢出
为什么 Python/JS 不做 TCO？ Python 的 BDFL 认为尾调用优化会让调试时的 stack trace 丢失关键帧。Lua 的取舍相反——它重视运行时效率而非调试便利性。这是两种语言定位的差异。

函数式编程模式

-- map：对每个元素应用函数
local function map(func, list)
    local result = {}
    for i, v in ipairs(list) do
        result[i] = func(v)
    end
    return result
end

local doubled = map(function(x) return x * 2 end, {1, 2, 3})
-- {2, 4, 6}

-- filter：保留满足条件的元素
local function filter(pred, list)
    local result = {}
    for _, v in ipairs(list) do
        if pred(v) then
            table.insert(result, v)
        end
    end
    return result
end

local evens = filter(function(x) return x % 2 == 0 end, {1, 2, 3, 4})
-- {2, 4}

-- reduce：累积计算
local function reduce(func, list, init)
    local acc = init
    for _, v in ipairs(list) do
        acc = func(acc, v)
    end
    return acc
end

local product = reduce(function(a, b) return a * b end, {2, 3, 4}, 1)
-- 24

基础练习

2

进阶挑战

用闭包实现一个 debounce(func, delay) 函数——在 delay 秒内多次调用只执行最后一次。这对于 UI 事件处理非常实用。

3

思考题

用尾递归实现一个有限状态自动机（FSA），解析简化版的 JSON 字符串。注意：必须是 proper tail call，不能栈溢出。

本章小结

  • 函数是一等公民：可赋值、传参、返回、匿名、运行时创建
  • 多返回值只有在调用位于表达式末尾时才全部保留，否则截断为 1 个
  • 闭包 = 函数 + upvalue（捕获的外部局部变量）。upvalue 是共享引用
  • 循环中创建闭包要创建局部快照，否则所有闭包指向同一个变量
  • Proper tail call 是语言规范保证的，无限尾递归不会栈溢出
