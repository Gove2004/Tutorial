Buff 与 Debuff 系统

状态是战斗深度的核心：药水、饰品、命中附带、持续伤害，都通过 Buff 系统实现。

Buff 不只是增益图标 很多新手把 Buff 理解成"屏幕上那排小图标"，但真正重要的是：它是一套完整的持续效果系统，能在玩家、敌怪、NPC 身上持续运行逻辑。

ModBuff 的定位

在 tModLoader 里，ModBuff 负责定义一个状态效果的类型、显示、持续逻辑和清除条件。它本身不关心"怎么触发"——触发通常由药水、饰品、弹幕命中、敌怪攻击等外部逻辑负责。

ModBuff

ModBuff 描述一个状态效果：它叫什么、图标是什么、是增益还是减益、存在期间会持续做什么。

创建一个基础增益 Buff

using Terraria;
using Terraria.ModLoader;

namespace ExampleMod.Content.Buffs;

public class RageBuff : ModBuff
{
    public override void SetStaticDefaults()
    {
        Main.buffNoSave[Type] = false;
        Main.buffNoTimeDisplay[Type] = false;
    }

    public override void Update(Player player, ref int buffIndex)
    {
        player.GetDamage(DamageClass.Generic) += 0.10f;
        player.GetCritChance(DamageClass.Generic) += 5f;
    }
}

这个 Buff 的核心在 Update 方法：只要玩家身上带着这个 Buff，这段逻辑就会持续执行。也就是说，增益不是"挂上去就结束"，而是"挂上去后持续修改玩家属性"。

SetStaticDefaults 里的常见配置

字段作用常见值
Main.buffNoSave[Type]是否在保存世界时清除debuff 通常为 true
Main.buffNoTimeDisplay[Type]是否隐藏时间显示无限时 Buff 通常为 true
Main.debuff[Type]是否是 Debuff减益为 true

Debuff：负面状态的实现

Debuff 和 Buff 在代码结构上几乎一样，最大的区别在于 Main.debuff[Type] = true 这一行，以及它通常带来负面效果。

using Terraria;
using Terraria.ModLoader;

namespace ExampleMod.Content.Buffs;

public class BurnDebuff : ModBuff
{
    public override void SetStaticDefaults()
    {
        Main.debuff[Type] = true;
        Main.buffNoSave[Type] = true;
        Main.buffNoTimeDisplay[Type] = false;
    }

    public override void Update(Player player, ref int buffIndex)
    {
        player.GetModPlayer().isBurning = true;
    }

    public override void Update(NPC npc, ref int buffIndex)
    {
        npc.GetGlobalNPC().isBurning = true;
    }
}

这里出现了两个 Update 重载：一个针对玩家，一个针对 NPC。也就是说，同一个 Debuff 可以同时影响玩家和敌怪，只要你分别实现对应逻辑。

注意 持续伤害类 Debuff 通常不直接在 Update 里扣血，而是通过 ModPlayer 或 GlobalNPC 的生命值更新钩子实现，避免和原版生命恢复机制冲突。

持续掉血 Debuff 的正确写法

如果你想做一个"中毒每秒扣 X 血"的效果，最好通过 UpdateBadLifeRegen 实现，而不是直接在 Update 里 player.statLife -= X。

using Terraria;
using Terraria.ModLoader;

namespace ExampleMod.Common.Players;

public class BurnDebuffPlayer : ModPlayer
{
    public bool isBurning;

    public override void ResetEffects()
    {
        isBurning = false;
    }

    public override void UpdateBadLifeRegen()
    {
        if (isBurning)
        {
            if (Player.lifeRegen > 0)
                Player.lifeRegen = 0;

            Player.lifeRegenTime = 0;
            Player.lifeRegen -= 16;
        }
    }
}

这个写法让持续伤害和原版生命恢复系统协调工作，而不是两套逻辑打架。lifeRegen 负值会让玩家持续掉血，数值越负掉得越快。

点击展开：为什么要用 lifeRegen 而不是直接扣血

因为 Terraria 的生命恢复系统本身很复杂，涉及自然恢复、药水恢复、装备恢复、Buff 恢复等多个来源。如果你直接扣 statLife，很可能和其他恢复机制产生时序冲突，导致掉血忽快忽慢。用 lifeRegen 就能让你的持续伤害融入原版系统。

如何触发 Buff

Buff 本身不会凭空出现，触发方式通常有几种：

1. 药水物品触发

Item.buffType = ModContent.BuffType<Content.Buffs.RageBuff>();
Item.buffTime = 60 * 60 * 5; // 5 分钟

2. 饰品持续触发

public override void UpdateAccessory(Player player, bool hideVisual)
{
    player.AddBuff(ModContent.BuffType<Content.Buffs.SpeedBuff>(), 2);
}

3. 弹幕命中触发

public override void OnHitNPC(NPC target, NPC.HitInfo hit, int damageDone)
{
    target.AddBuff(ModContent.BuffType<Content.Buffs.BurnDebuff>(), 180);
}

4. 敌怪攻击玩家触发

public override void OnHitPlayer(Player target, Player.HurtInfo hurtInfo)
{
    target.AddBuff(ModContent.BuffType<Content.Buffs.PoisonDebuff>(), 240);
}

你会发现，触发 Buff 的核心 API 就是 AddBuff(buffType, duration)。第一个参数是 Buff 类型，第二个是持续时间（单位：帧）。

Buff 图标与本地化

Buff 的图标通常是 18x18 或 32x32 的 PNG，命名规则与类名一致。放在与代码相同的目录结构下，tModLoader 会自动加载。

如果图标不显示，优先查：

  • 文件名是否与类名完全一致（包括大小写）。
  • 图片是否放在正确的命名空间对应路径下。
  • 图片格式是否为 PNG。
图标设计建议 Buff 图标要在很小的尺寸下一眼看出是什么。别做太复杂的渐变和细节，优先用明确的形状、颜色和符号。

Buff 能否叠加、能否被移除

很多 Buff 不应该叠多层，也不应该被玩家用"右键点掉"。这些都可以在 SetStaticDefaults 里配置：

public override void SetStaticDefaults()
{
    Main.debuff[Type] = true;
    BuffID.Sets.NurseCannotRemoveDebuff[Type] = true;
}

NurseCannotRemoveDebuff 阻止护士 NPC 清除这个 Debuff。这对某些剧情或机制 Debuff 很有用，避免玩家用护士绕过设计意图。

NPC 端的 Buff 效果

敌怪也可以带 Buff。比如你想让某个敌怪被点燃后行动变慢，就需要在 GlobalNPC 或对应 ModNPC 里处理：

using Terraria;
using Terraria.ModLoader;

namespace ExampleMod.Common.GlobalNPCs;

public class BurnDebuffNPC : GlobalNPC
{
    public override bool InstancePerEntity => true;
    public bool isBurning;

    public override void ResetEffects(NPC npc)
    {
        isBurning = false;
    }

    public override void UpdateLifeRegen(NPC npc, ref int damage)
    {
        if (isBurning)
        {
            if (npc.lifeRegen > 0)
                npc.lifeRegen = 0;

            npc.lifeRegen -= 20;
            damage = 5;
        }
    }
}

这个写法和玩家端类似，只是钩子名字从 UpdateBadLifeRegen 变成了 UpdateLifeRegen。

条件触发与视觉反馈

有些 Buff 不仅改数值，还会有粒子、光效、音效。你可以在 Update 里加 Dust.NewDust 或播放声音，让 Buff 的存在更明显。

public override void Update(Player player, ref int buffIndex)
{
    player.GetDamage(DamageClass.Generic) += 0.15f;

    if (Main.rand.NextBool(10))
    {
        Dust.NewDust(player.position, player.width, player.height, DustID.FireworkFountain_Red);
    }
}

但别过度。如果每一帧都生成 Dust，画面会非常吵，性能也会受影响。通常用 Main.rand.NextBool 控制概率。

常见坑

  • 忘记在 ModPlayer/GlobalNPC 的 ResetEffects 里重置状态标记，导致 Buff 效果消失后仍在生效。
  • 直接在 Update 里扣 statLife，和原版恢复机制冲突。
  • Buff 图标命名或路径错误，导致显示为空白。
  • Debuff 没有设置 Main.debuff[Type] = true，导致被当成增益处理。
排查建议 Buff 问题通常分两层：触发问题（是否真的挂上了）和效果问题（挂上了但没生效）。先用 player.HasBuff 确认触发，再查 Update 逻辑。
